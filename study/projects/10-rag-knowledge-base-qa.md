## 4. 在线问答设计

设计在线流程：

```text
用户问题
  -> 权限识别
  -> 查询改写
  -> 检索
  -> Rerank
  -> Prompt 组装
  -> 模型回答
  -> 引用与校验
```

说明每一步的作用。

## 5. RAG Prompt 设计

要求 Prompt 包含：

- 角色。
- 只能基于资料回答。
- 资料不足时拒答。
- 必须引用来源。
- 输出结论、依据、不确定点。

## 6. 权限设计

设计三个角色：

- 普通员工。
- 部门管理员。
- 系统管理员。

说明每个角色能检索哪些文档。

【重点】权限过滤必须发生在检索阶段。

## 8. 故障排查任务

分析下面问题可能原因：

1. 用户问的问题明明文档里有，但系统回答不知道。
2. 检索结果有正确文档，但模型回答错。
3. 模型回答没有引用来源。
4. 普通用户看到了无权限文档。
5. 文档更新后系统仍回答旧内容。
6. 回答很长但没有重点。
