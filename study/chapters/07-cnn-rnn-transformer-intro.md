# 第 7 课：CNN、RNN 与 Transformer 入门

## 0. 本课阅读方式

本课介绍三类重要神经网络结构：

```text
CNN：擅长处理局部空间结构，常见于图像。
RNN：擅长处理序列，早期常用于文本和时间序列。
Transformer：擅长并行建模序列关系，是现代大模型基础。
```

本课不要求你从零实现复杂网络，而是理解：

```text
它们分别解决什么问题？
输入数据长什么样？
核心结构是什么？
为什么 Transformer 取代 RNN 成为大模型主流？
```

【核心】CNN、RNN、Transformer 是深度学习中的三种重要结构思想，不只是三个 API 名称。

### 0.1.2 CNN

CNN 是 Convolutional Neural Network，卷积神经网络。

它擅长处理图像，因为图像具有局部空间结构：

```text
相邻像素之间关系很重要。
局部纹理可以组合成更高级图案。
```

【核心】CNN 通过卷积核提取局部特征。

### 核心讲解（课程模式 · 0.1.2 · 2026-05-14）

把上一节的「归纳偏置」落到 CNN 这一句：**同类局部模式在不同空间位置反复复用**。全连接层把每个像素当成独立坐标，图像稍一平移语义不变但输入向量整段变化；CNN 改用 **在空间上滑动** 的同一个小模版（卷积核）去读局部小块，再在深层把局部组合成部件与物体。

这一节你只记四件事就够：

```text
1) CNN ≈ 「局部窗口上的线性加权」（卷积运算）层层堆叠。
2) 同一套核对整幅图扫过去 → 权重共享 → 参数远小于把所有像素拉开做全连接。
3) 学到的不是 Java 手写 if/else「找边」，而是一组可由损失函数反传的实数矩阵。
4) 真实工程里常有「多输入通道」（RGB、或上一层多个特征图堆叠）；本课先有单通道心智模型，
   到 0.1.5 会看到「一张图会变成很多张特征图」。
```

### 0.1.3 Convolution 卷积

卷积是用一个小窗口在图像上滑动，提取局部模式。

卷积核可以学习：

- 边缘。
- 纹理。
- 角点。
- 局部形状。

【重点】卷积层不是手写边缘检测规则，而是通过训练学习卷积核参数。

### 核心讲解（课程模式 · 0.1.3 · 2026-05-14）

在 CNN 讲义里常说的「二维卷积」，工程实现多数是 **离散互相关（cross-correlation）**：核对齐 patch 左上角，**逐项相乘再求和**。数学上的「严格卷积」还会把核对折再滑动；你读论文若看到记号差异，先在脑子里统一成：**一个小窗口在平移中取局部加权和**。

本小节再记三件事：

```text
1) 单层一次「扫完」的流程：对每个输出格子，选一个与核同样大的输入区域 → 加权求和
   （再经激活函数等，本课可先略）。
2) 三个旋钮改变输出尺寸：stride（步幅）、padding（边缘补零）、kernel（核边长）。
3) 「valid + stride=1 + 无 pad」就是你跑过的脚本：输出每条边变小；若你希望高宽不降，
   可用 padding（或稍后再用别的结构），下一脚本用公式推演。
```

**输出长度（单列 / 单行；与 PyTorch Conv2d 在 dilation=1 时的常见推导一致）**：

```text
out_len = floor((L + 2*P - K) / S) + 1
```

此处 `L` 为输入边长，`K` 核边长，`P` 每侧对称 padding，`S` 为 stride。**前提**是括号内结果为 **非负** 且与你的框架取整约定一致（本课用整数推演即可）。

**跟练（约 10 分钟）**：运行 **`ai-learning/07-cnn-rnn-transformer-intro/scripts/cnn_output_spatial_formula_demo.py`**，对照三组数字感受 **stride / padding** 如何把 **6×6** 的输出变回 **`8`** 或与 **池化前的堆叠深度**挂钩（脚本只算术，不修图）。

```bash

```

### 0.1.4 Kernel / Filter 卷积核

卷积核是一个小矩阵。

例如：

```text
3 x 3
5 x 5
```

它在图像上滑动，对局部区域做加权计算。

【核心】卷积核是 CNN 中可学习的局部特征检测器。

### 核心讲解（课程模式 · 0.1.4 · 2026-05-14）

把 **0.1.3** 的「运算」具体到 **一样东西**：卷积核就是 **一小块与 patch 对齐的权重张量**。单通道情形下你把它看成 **`K×K` 实数矩阵**即可；训练中通过反向传播改动这些数，使某项损失下降。

本节再记三件具体事：

```text
1) 一个核在整个空间上重复使用 → 「权重共享」，这是 CNN 参数远少于全连接的关键之一。
2) 一层里通常不止一个核：第 i 个核负责一种（可学习的）局部响应模式，输出第 i 张特征图。
3) 深层的「核」读的不一定是 RGB，而是上一层的特征图拼成的多通道体；那是 0.1.5 与工程里的 in_channels / out_channels。
```

**与上节脚本关系**：`cnn_valid_conv_numpy_demo.py` 里写死的 **`3×3`** 是一段 **常量核**；等价于未经训练的「手写局部检测器」，用来对上 **乘法—累加** 有体感。真实网络里的核 **`requires_grad`**，由 **`loss.backward()`**（第 6 课骨架）迭代更新。

### 拓展阅读

- **下一节 `### 0.1.5 Feature Map`**：从「多张输出」讲到 **低级 / 高级**语义直觉。  
- **反向传播直觉**：某项损失对输出的敏感会一路乘链式法则传到每个核权重上——把核想成「可旋钮的局部模版」足够。

### 联想扩展

- **Java 类比**：若把 **单层卷积**想成：`for each 输出坐标` → `weightedSum(patch, fixedKernelWeights)`——**不同之处**是全图共用 **同一份 Kernel 对象的权重向量**，梯度对 **所有出现位置**的贡献会 **相加**后再更新（实现里已向量化完成）。

### 0.1.5 Feature Map 特征图

卷积层输出的结果叫特征图。

低层特征图可能表示：

- 边缘。
- 颜色变化。
- 纹理。

高层特征图可能表示：

- 眼睛。
- 轮廓。
- 物体部件。

### 核心讲解（课程模式 · 0.1.5 · 2026-05-14）

**特征图（feature map）**在工程里指：**某一层卷积（或类似局部线性算子）在空间上「扫完一遍」之后得到的那张二维响应场**。输入若是 RGB，第一层读的是 **`3×H×W`**；该层若有 **`C_out`** 个核并行扫图，就得到 **`C_out` 张**同尺寸的（或按 0.1.3 公式缩放后的）特征图，其中 **深度维等于 `C_out`**，语义是「并行检测几种局部模式」。

本节记四件事：

```text
1) 张量形状口语：常记 ``(N, C, H, W)``——本课可先只盯单样本时的 ``(C, H, W)``。
2) 第 i 张特征图 ≈ **第 i 个卷积核**在整幅输入上滑出来的「激活图」。
3) 「低层像边缘、高层像部件」是 **经验直觉**：帮助建立层次化表征的心智模型，不是严格可证的分类定义。
4) 深网络里常见 **通道变多、单张图空间略缩小**（靠 stride / pooling 等），像 **从细网格的局部响应 → 更粗网格的语义组合**。
```

**自测**：若 ``C_in=64``、`C_out=128`，且空间高宽未被改变，输出特征图的 **通道维**应是几？——**128**（与核个数一致）。

### 拓展阅读

- **下一小节 `### 0.1.6`**：在 **不下调参数量**（通常无权重）的前提下 **压空间分辨率**。  
- **ResNet / 现代 CNN**：有的块用 **stride>1 的卷积**代替部分池化角色；先把 **池化的直观**抓住即可。

### 联想扩展

- **与 0.1.4 勾连**：**多核**是 **多张特征图**的直接原因；不要再把「RGB 三通道」与「第 40 通道」混成同一语义——前者是输入约定，后者是 **学出来的抽象特征索引**。  
- **Java 类比**：可把 **固定 H×W 上的一层 C 张特征图**想成 **C 个并行的二维 `double[][]` 面板**，面板之间通过下一层核的 **`in_channels`** 再耦合。

### 0.1.6 Pooling 池化

池化用于压缩特征图大小。

常见：

- Max Pooling
- Average Pooling

作用：

- 降低计算量。
- 提高对小范围位置变化的鲁棒性。
- 保留重要特征。

### 核心讲解（课程模式 · 0.1.6 · 2026-05-14）

**池化（pooling）**在局部窗口内做 **固定的汇聚规则**（常见 **Max** 或 **Average**），一般 **不学参数**。它吃一张（或多通道上的同位置窗口）特征响应，吐一个标量，因此 **空间分辨率往往下降**（例如 **`2×2` 窗口、`stride=2`** → 高宽各约减半）。

对比卷积：

```text
- 卷积：窗口内与可学习核做加权和 —— 参数多、表达细。
- 池化：窗口内取 max 或 mean —— 常无参数、偏「抽象与抑噪」。
```

**Max Pooling** 直觉：**这一小块里谁最「响」就信谁**，对微小平移更钝感；**Average Pooling** 直觉：**把局部能量平均掉**，更平滑、有时少丢背景但锐度弱。许多经典视觉网络里 **Max 更常见**，但具体任务会换设计。

```bash
python ai-learning/07-cnn-rnn-transformer-intro/scripts/pooling_2x2_stride2_numpy_demo.py
```

**自测**：输入 **`4×4`**、**`2×2` 池化**、**`stride=2`**、**`padding=0`**，输出空间边长是几？——**2**。

### 0.1.7 RNN

RNN 是 Recurrent Neural Network，循环神经网络。

它用于处理序列数据。

序列数据特点：

```text
当前元素的含义和前后顺序有关。
```

例如：

- 文本。
- 时间序列。
- 语音。
- 用户行为序列。

【核心】RNN 通过隐藏状态保存历史信息。

### 核心讲解（课程模式 · 0.1.7 · 2026-05-15）

第 6 课若有印象：**多层全连接 / 卷积** 吃的是 **固定形状** 的一批输入；而 **序列** 的长度随样本变化（句子长短、日志条数、采样点数），且 **先后顺序改变语义**。RNN 族的做法是同一个小网络 **沿时间维复用**：每一步喂一个向量 **x_t**，内部维护一个 **隐藏状态** **h_t**，作为「压缩后的历史摘要」传向下一步。

本小节抓住三件事即可：

```text
1) 「循环」= 同一套参数在每个时间步重算；不是为每个 t 建一套独立权重。
2) 直觉上 h_t 携带「看过 x_1..x_t 之后」的信息；下一步读 x_{t+1} 时两者再融成新的 h。
3) 与 CNN 对比：CNN 的归纳偏置是 **空间局部**；RNN 的是 **时间因果与共享变换**（实现上当然可有双向变体，本课先记单向心智模型）。
```

CNN 把一整张图并行扫完；经典 RNN 则是 **时间上前向顺序**（一步依赖上一步），因此 **长序列训练时早期并行度差**，这也是后来 Transformer 在「长上下文、可并行」上反超的一个工程原因——**0.1.10** 再对照。

### 0.1.8 Hidden State 隐藏状态

隐藏状态是 RNN 记忆历史信息的向量。

每一步输入后，RNN 更新隐藏状态：

```text
当前输入 + 上一步隐藏状态 -> 新隐藏状态
```

【重点】隐藏状态是 RNN 处理序列的核心。

### 核心讲解（课程模式 · 0.1.8 · 2026-05-15）

**隐藏状态（hidden state）**就是上面说的 **h_t**：一步算完，只把这个长度固定的向量交给下一步；**不把**整个历史序列原样堆进内存（那是朴素的「上下文窗口爆炸」）。工程上你会看到 **BPTT（随时间反传的截断）**、**梯度消失/爆炸** 等词——直觉上：tanh/sigmoid 反复复合后，**非常远的 x_k 对当前 h_t 的梯度**可能趋近于零（消失）或爆掉（爆炸），所以才有 **LSTM/GRU（0.1.9）** 与后来的 **Attention**。

本小节记住递推骨架（与演示脚本一致）：

```text
h_t = tanh(W_xh * x_t + W_hh * h_{t-1} + b)

x_t：当前步输入（例如 embedding 后的 token）
h_{t-1}：上一步隐藏向量；第一步常用全零初始化 h_0
W_xh, W_hh, b：跨时间步共享的可学习参数
```

**读输出**：很多时候还会再有 **V * h_t + c** 得到每一步的 logits（分类、下一个词概率等）；本课演示只做到 **h_t**，避免一次塞太多符号。

**跟练**：仍用 **`vanilla_rnn_numpy_demo.py`**，这次专注看 **`h_t L2` 与 `h_t` 向量** 逐步怎么变；可与自测「全零输入」对照 **饱和**（tanh 接近 ±1）直觉。

### 联想扩展

- **坑**：把「隐藏维」与「序列长度 T」混成一个数——前者是 **向量长度**，后者是 **步数**。

### 0.1.9 LSTM / GRU

普通 RNN 容易遇到长距离依赖问题。

LSTM 和 GRU 是改进结构，用门控机制控制信息保留和遗忘。

适合：

- 较长文本。
- 时间序列。
- 需要记忆历史上下文的任务。

【重点】LSTM/GRU 曾是 NLP 主流结构，但大模型时代主要由 Transformer 接替。

### 核心讲解（课程模式 · 0.1.9 · 2026-05-15）

**普通（香草）RNN** 把历史挤进单个 **h_t**，反复乘 **W_hh** 与非线性，**梯度反传**时容易 **连乘衰减（消失）** 或 **放大（爆炸）**，长句子远处 token 很难有效影响当前步。工程上已有 **截断 BPTT、梯度裁剪、好的初始化**，但 **结构层**上更直接的修补是 **门控**：显式决定 **哪些信息该沿时间保留、哪些该更新**。

**LSTM** 拆成两条线：**细胞状态 c_t**（慢变、像「传送带」）与 **隐藏状态 h_t**（对外接口）。直观记四个门的名相（实现细节各家略异，本课记心智模型即可）：

```text
f（forget）：旧 c 里多少保留
i（input）：新候选写入多少
g（candidate）：写入 c 的候选内容（常配合 tanh）
o（output）：从新的 c 里读出多少到 h
```

**GRU** 把门控 **合并得更紧**：常用 **更新门 z** 在「上一步 h」与「候选新内容」之间 **插值**，**复位门 r** 控制多大程度上忘掉过去再算候选。参数通常比同宽 LSTM 少，训练有时更快，**表达能力与数据量任务强相关**，没有绝对的「谁永远更好」。

```bash

```

**自测**：LSTM 若 **f 全为 1、i 全为 0**，一步里 **c** 相对上一步会怎样？——**不变**（只剩 f*c，i*g 为 0）。这是对「长程拷贝」友好的极端情况，训练中门值由数据学出来。

### 0.1.10 Transformer

Transformer 是现代大模型的核心结构。

它不依赖循环逐步处理序列，而是通过 Attention 机制直接建模序列中任意位置之间的关系。

【核心】Transformer 是大模型的基础架构。

### 核心讲解（课程模式 · 0.1.10 · 2026-05-15）

**Transformer** 块（先说 Encoder 侧心智模型）把「处理一个 token」从 **依赖上一步隐藏状态** 改成：**在这一层里，让每个位置都能直接与其它位置建联**（通过 **Attention**），再经 **前馈子层（FFN）** 做逐位置非线性。堆叠多层后，**感受野**随层加深而扩大，**不再**靠 RNN 的 **T 步串行** 把信息一步步传来。

本小节先立三块：**(1) 并行**：训练一步内许多矩阵乘可对齐批量算；**(2) 长程**：单次 Attention 可在 **受限上下文长度内** 连任意两端（代价是 **O(T²)** 对长度的开销，长上下文要工程折中）；**(3) 位置**：模型本身对 token 集合 **置换不变**，须加 **位置编码（0.1.15）** 才能利用词序。

【对照 0.1.7～0.1.9】：**RNN 是一步一步读「过去」**；**Self-Attention 是一层内所有位置互看**（再 masked 的变体用于解码器）。二者都能建模依赖，但 **归纳偏置与计算图形状** 完全不同。


### 0.1.11 Attention 注意力机制

Attention 的直觉是：

```text
处理当前位置时，模型应该关注序列中哪些其他位置？
```

例如句子：

```text
小明把书放进书包，因为它很重。
```

模型需要判断“它”指什么。

Attention 可以让模型在处理“它”时关注“书”。

### 核心讲解（课程模式 · 0.1.11 · 2026-05-15）

**Attention（注意力）**在工程里常被说成「软寻址」：**当前位置**去翻一本只有 **T 页** 的字典（序列里 **T 个位置**），每页给个 **权重**，再按权重 **加权求和** 得到一个向量。与硬编码 `if 位置==i` 不同，权重 **连续、可导**，才能用梯度学习。

**Q（Query）**可理解为「我在查什么」；**K（Key）**是各条记忆被匹配的 **键**；二者相似则权重大。**V（Value）**才是被聚合进最终向量的 **内容**。在 **缩放点积** 形式里，相似度常用 **Q K^T / sqrt(d_k)** 再 **逐行 softmax**，再去 **乘 V**——你在 **0.1.10** 已跑过同一套算术。

**Cross-Attention（交叉注意力）**：Q 来自 **序列 A**（如解码器当前步），K、V 来自 **序列 B**（如编码器输出）——翻译、多模态里常见。**Self-Attention**：Q、K、V 都来自 **同一序列**（见 **0.1.12**）。

**跟练（约 10 分钟）**：先复习 **`scaled_dot_product_attention_numpy_demo.py`**；再运行 **`self_attention_numpy_demo.py`**，看 **Q、K、V 由 X 经三个 W 映射** 后再出 **权重与 context**。

```bash
python ai-learning/07-cnn-rnn-transformer-intro/scripts/self_attention_numpy_demo.py
```

**自测**：若把 **W_K** 整体 **乘以 2** 而 **Q、V 不变**，softmax 前的 **scores** 会怎样？——**整体放大**，softmax 更 **尖**（更关注极少数位置）；除以 **sqrt(d_k)** 正是为了缓解点积方差随维度变大。

### 0.1.12 Self-Attention 自注意力

Self-Attention 是序列内部各位置之间互相关注。

每个 token 都可以看其他 token，并计算相关性。

【核心】Self-Attention 让 Transformer 能建模长距离依赖。

### 核心讲解（课程模式 · 0.1.12 · 2026-05-15）

**Self-Attention（自注意力）**就是：**对同一个长度为 T 的向量序列 X**，用 **三个可学习线性投影** 得到 **Q、K、V**，再做 **0.1.10** 的缩放点积注意力。于是 **每一层**里，**任意两端**（在长度与显存允许范围内）可以直接通过 **高权重** 相连，而不必像 RNN 那样链式传递 **T 次**。

**因果掩码（causal mask）**：**Decoder-only**（如 GPT）在训练生成时，位置 **i** 不应「看见」位置 **j > i** 的未来 token。做法是在 softmax 前给 **禁止连接的位置** 加 **-inf**（或极大负数），使对应权重为 0。Encoder 双向编码通常 **不**对完整输入加因果掩码；**前缀填充 + 变长**另有 **padding mask**，属实现细节。

**一层之后**：通常还有 **残差** `X + SubLayer(...)` 与 **LayerNorm**（Post-LN 或 Pre-LN 依架构而定），再进 **FFN**；本课先抓住 **Attention 子层** 即可。

**跟练**：与 **0.1.11** 共用 **`self_attention_numpy_demo.py`**；可手动改小 `T`、打印某一行 **weights**，对照 **哪一列更大**。

**自测**：自注意力里 **输出 context 的第 t 行** 一定是 **V 的行的线性组合**，组合系数是 **weights 的第 t 行**；若只想让位置 t **只看自己**，权重行应长什么样？——**one-hot** 在第 t 列为 1（极端情况，一般学习不会锁死）。

### 0.1.13 Token

Token 是文本被模型处理的基本单位。

它可能是：

- 一个词。
- 一个子词。
- 一个字符片段。
- 一个标点。

【重点】大模型不是直接处理原始字符串，而是先把文本切成 token。

### 核心讲解（课程模式 · 0.1.13 · 2026-05-15）

**Token**是模型眼里 **「可计数的离散符号」**：一串 **整数 id** 的下标对应物。它 **不等于** 自然语言的「词」——可能是：**整词**、**子词（BPE、SentencePiece）**、**字/字符**、**特殊符号**（如 `[CLS]`、`</s>`），甚至 **补丁块**（视觉 Transformer）。**分词方案 + 词表**由训练前确定，推理时必须 **与训练时一致**，否则 id 对不齐。

你要记住三件事：

```text
1) 计费与上下文窗口通常按 token 计，不是按「汉字/英文单词」字面数。
2) 同一句话在不同分词器下 token 数可差很多；工程上要面对 max_context、成本与延迟。
3) 特殊 token 常用来打包任务信息（起始、结束、填充、角色区分）；具体集合看各模型文档。
```

```bash
python ai-learning/07-cnn-rnn-transformer-intro/scripts/token_embedding_lookup_numpy_demo.py
```

**自测**：若词表里 **删掉某词**、但仍用原句跑分词，id 应落到哪？——演示里应走 **`<UNK>`**（真实系统需统一约定未知词策略）。

### 0.1.14 Embedding

Embedding 把 token 转成向量。

Transformer 处理的是向量序列，不是原始文本。

```text
文本 -> token -> embedding -> Transformer
```

### 核心讲解（课程模式 · 0.1.14 · 2026-05-15）

**Embedding**把 **稀疏的离散 id** 映到 **稠密向量**（一节 **可学习** 的查表：`nn.Embedding(num_embeddings, embedding_dim)` 的本质是 **权重矩阵** **E**，第 **i** 行是 id=i 的向量）。这样 **Attention、FFN** 才能做矩阵乘。**d_model（或 d_embed）** 是向量维度，与 **0.1.12** 里 **X** 的列数对齐；真实模型里 **词嵌入维** 常与 **隐藏维** 一致或通过投影统一。

直觉：**one-hot** 再乘 **E** 等价于 **取出 E 的第 i 行**；实现上绝不真的会造巨型 one-hot（只 **按索引取值**）。训练时 **E** 与上面各层 **一起反传**，语义相近的 token 在向量空间里会被拉近（受任务与数据驱动，无魔法）。

**跟练**：与 **0.1.13** 共用 **`token_embedding_lookup_numpy_demo.py`**；把输出 **X** 代入心智流：**再接位置编码 → 再进 Transformer 块**（下一小节）。

**自测**：若 **vocab_size=10000**、**d_embed=512**，**E** 有多少个标量参数？——**10000×512**（仅嵌入层）。

### 拓展阅读

- **下一小节 `### 0.1.15`**：为何光有 embedding **不够**，还要 **位置信息**。  
- **共享权重**：有的架构 **输入嵌入与输出 softmax 权重共享**（tie weights），减少参数。  
- **多模态**：图像块、音频帧也可 **embedded** 后进类 Transformer 栈。

### 联想扩展

- **与查表缓存**：生产上 **E** 常驻 GPU；超长序列占的是 **激活** 与 **KV cache**，不只是 embedding。  
- **冷启动**：新词若不在词表，会 **UNK** 或 **回退子词**——线上要监控 **UNK 率**。

### 0.1.15 Positional Encoding 位置编码

Transformer 本身没有 RNN 的顺序处理机制，因此需要位置编码告诉模型 token 顺序。

【重点】没有位置信息，Transformer 很难区分词序。

### 核心讲解（课程模式 · 0.1.15 · 2026-05-15）

**Self-Attention** 对 token 置换在结构上是对称的：若只换 **embedding 的行序** 而不告知 **谁在第几个位置**，则 **QK^T** 里仍会建立联系，但 **「先读后写」** 等顺序语义会丢。因此要往每个位置注入 **位置信息**。常见两类：

```text
1) 固定函数式（正弦/余弦）：只依赖 (pos, dim)，无额外可学习参数；原论文可外推到更长 pos（实践上仍受训练分布约束）。
2) 可学习位置嵌入：与 token embedding 一样是查表，「第 pos 行」可训练；外推与插值策略要单独设计。
```

工程上还涌现 **RoPE（旋转位置）**、**ALiBi（偏置）** 等；本课先掌握 **「embedding + 位置项 → 再进子层」**。最简图画：**X = X_emb + PE**（或 **concat 再投影**，依实现而定）。

**跟练（约 10 分钟）**：运行 **`sinusoidal_positional_encoding_numpy_demo.py`**，核对 **pos=0/1** 的 **PE** 行与 **加减恒等式**。

```bash
python ai-learning/07-cnn-rnn-transformer-intro/scripts/sinusoidal_positional_encoding_numpy_demo.py
```

**自测**：**d_model** 固定时，**PE** 的第 **t** 行是否 **仅由 t 决定**、与句子里是哪个词无关？——**是**（正弦式里不含 token id）；语义仍来自 **X_emb**。

### 0.1.16 Encoder / Decoder

Transformer 常见结构包括：

- Encoder：理解输入。
- Decoder：生成输出。

模型类型：

| 类型 | 示例任务 |
|---|---|
| Encoder-only | 文本分类、理解 |
| Decoder-only | 文本生成、大语言模型 |
| Encoder-Decoder | 翻译、摘要 |

【核心】GPT 类模型通常是 Decoder-only Transformer。

### 核心讲解（课程模式 · 0.1.16 · 2026-05-15）

**Encoder**块：**双向**理解整段输入（在 **无因果掩码** 的前提下，每一位置可 attend 全序列），适合 **分类、检索向量、填空理解**。**Decoder**块：**自回归生成**，训练时常带 **因果掩码**（只看过去 token），推理时 **逐 token 延拓**。**Encoder-Decoder**：Encoder 压缩源序列，Decoder **交叉注意力**读 Encoder 输出——经典 **翻译、摘要**（如早期 seq2seq Transformer）。

产品线心智表（命名会随年代变体，抓结构即可）：

```text
- Encoder-only：如 BERT 系 ——「读得好」，生成要另接头。
- Decoder-only：如 GPT 系 ——「写得顺」，也是当前 **通用大模型** 最主流形态。
- Encoder-Decoder：如部分 T5、BART —— 明确的「源→目标」编解码分工。
```

【与前面小节串起来】：**Token + Embedding + 位置编码** 形成输入；**Encoder 栈**或 **Decoder 栈**里重复 **Attention + FFN + 残差/Norm**；**损失函数与任务头**决定是 **LM 下一词** 还是 **分类交叉熵** 等。

**跟练**：用本课已跑过的脚本在纸上画一条数据流：`token id` → `E` 得 `X_emb` → `+ PE` → **一层** `Self-Attention`（可用 **`self_attention_numpy_demo.py`** 代替「一层」）——不必一次画 96 层。

**自测**：**Decoder-only** 训练长为 **T** 的句子时，**最末位置**的 query 在 **不加掩码**时会看到什么？——会 **看到未来**，与自回归训练目标 **冲突**；故必须 **因果 mask** 或等价机制。

## 2. 三类结构对比

| 结构 | 擅长数据 | 核心思想 | 典型应用 |
|---|---|---|---|
| CNN | 图像、局部空间结构 | 卷积提取局部特征 | 图像分类、目标检测 |
| RNN | 序列 | 隐藏状态传递历史 | 文本、语音、时间序列 |
| Transformer | 序列和多模态 | Attention 建模全局关系 | 大模型、翻译、摘要 |

## 3. CNN 详解

CNN 利用图像的局部性：

```text
边缘 -> 纹理 -> 部件 -> 物体
```

网络越深，特征越抽象。

【工程经验】CNN 仍然是很多视觉任务的重要基础，即使现在视觉 Transformer 也很流行。

## 4. RNN 详解

RNN 逐步读取序列：

```text
x1 -> h1
x2 + h1 -> h2
x3 + h2 -> h3
```

问题：

- 训练难。
- 长距离依赖弱。
- 难以并行。

这些问题推动了 Transformer 的流行。

## 5. Transformer 详解

Transformer 用 Self-Attention 让每个 token 直接关注其他 token。

优势：

- 并行能力强。
- 长距离依赖更好。
- 可扩展到大规模数据和参数。

【核心】大语言模型的核心能力建立在 Transformer 可扩展性上。

## 6. 从文本到 Transformer

流程：

```text
原始文本
  -> Tokenization
  -> Token IDs
  -> Embedding
  -> Positional Encoding
  -> Transformer Blocks
  -> 输出概率或表示
```

【重点】模型真正处理的是数字向量，不是原始文字。

## 7. 常见误区

1. 以为 CNN 只能用于图片。它也可用于一维信号。
2. 以为 RNN 已经完全没用。它仍可用于小型序列任务。
3. 以为 Transformer 不需要位置信息。
4. 以为 Attention 就是简单关键词匹配。
5. 以为大模型只靠结构，忽略数据和训练规模。

## 8. 代码辅助示例

### 8.1 CNN 层形状直觉

```python
import torch
from torch import nn

x = torch.randn(8, 3, 32, 32)
conv = nn.Conv2d(in_channels=3, out_channels=16, kernel_size=3)
y = conv(x)

print(y.shape)
```

含义：

```text
8：batch size
3：输入通道
32x32：图像大小
16：输出特征图通道数
```

### 8.2 Transformer 输入直觉

```python
import torch
from torch import nn

embedding = nn.Embedding(num_embeddings=10000, embedding_dim=128)
token_ids = torch.tensor([[1, 23, 456, 78]])
x = embedding(token_ids)

print(x.shape)
```

含义：

```text
token id -> embedding 向量
```
