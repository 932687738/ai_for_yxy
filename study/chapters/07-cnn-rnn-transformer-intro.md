# 第 7 课：CNN、RNN 与 Transformer 入门

## 0. 本课阅读方式

本课介绍三类重要神经网络结构：

```text
CNN：擅长处理局部空间结构，常见于图像。
RNN：擅长处理序列，早期常用于文本和时间序列。
Transformer：擅长并行建模序列关系，是现代大模型基础。
```

本课不要求你从零实现复杂网络，而是理解：

```text
它们分别解决什么问题？
输入数据长什么样？
核心结构是什么？
为什么 Transformer 取代 RNN 成为大模型主流？
```

【核心】CNN、RNN、Transformer 是深度学习中的三种重要结构思想，不只是三个 API 名称。

### 0.1.2 CNN

CNN 是 Convolutional Neural Network，卷积神经网络。

它擅长处理图像，因为图像具有局部空间结构：

```text
相邻像素之间关系很重要。
局部纹理可以组合成更高级图案。
```

【核心】CNN 通过卷积核提取局部特征。

### 核心讲解（课程模式 · 0.1.2 · 2026-05-14）

把上一节的「归纳偏置」落到 CNN 这一句：**同类局部模式在不同空间位置反复复用**。全连接层把每个像素当成独立坐标，图像稍一平移语义不变但输入向量整段变化；CNN 改用 **在空间上滑动** 的同一个小模版（卷积核）去读局部小块，再在深层把局部组合成部件与物体。

这一节你只记四件事就够：

```text
1) CNN ≈ 「局部窗口上的线性加权」（卷积运算）层层堆叠。
2) 同一套核对整幅图扫过去 → 权重共享 → 参数远小于把所有像素拉开做全连接。
3) 学到的不是 Java 手写 if/else「找边」，而是一组可由损失函数反传的实数矩阵。
4) 真实工程里常有「多输入通道」（RGB、或上一层多个特征图堆叠）；本课先有单通道心智模型，
   到 0.1.5 会看到「一张图会变成很多张特征图」。
```

### 0.1.3 Convolution 卷积

卷积是用一个小窗口在图像上滑动，提取局部模式。

卷积核可以学习：

- 边缘。
- 纹理。
- 角点。
- 局部形状。

【重点】卷积层不是手写边缘检测规则，而是通过训练学习卷积核参数。

### 核心讲解（课程模式 · 0.1.3 · 2026-05-14）

在 CNN 讲义里常说的「二维卷积」，工程实现多数是 **离散互相关（cross-correlation）**：核对齐 patch 左上角，**逐项相乘再求和**。数学上的「严格卷积」还会把核对折再滑动；你读论文若看到记号差异，先在脑子里统一成：**一个小窗口在平移中取局部加权和**。

本小节再记三件事：

```text
1) 单层一次「扫完」的流程：对每个输出格子，选一个与核同样大的输入区域 → 加权求和
   （再经激活函数等，本课可先略）。
2) 三个旋钮改变输出尺寸：stride（步幅）、padding（边缘补零）、kernel（核边长）。
3) 「valid + stride=1 + 无 pad」就是你跑过的脚本：输出每条边变小；若你希望高宽不降，
   可用 padding（或稍后再用别的结构），下一脚本用公式推演。
```

**输出长度（单列 / 单行；与 PyTorch Conv2d 在 dilation=1 时的常见推导一致）**：

```text
out_len = floor((L + 2*P - K) / S) + 1
```

此处 `L` 为输入边长，`K` 核边长，`P` 每侧对称 padding，`S` 为 stride。**前提**是括号内结果为 **非负** 且与你的框架取整约定一致（本课用整数推演即可）。

**跟练（约 10 分钟）**：运行 **`ai-learning/07-cnn-rnn-transformer-intro/scripts/cnn_output_spatial_formula_demo.py`**，对照三组数字感受 **stride / padding** 如何把 **6×6** 的输出变回 **`8`** 或与 **池化前的堆叠深度**挂钩（脚本只算术，不修图）。

命令：

```bash

```

**自测**：输入 `L=32`，核 `K=3`，`P=1`，`S=2`，手算括号再与脚本核对。

### 0.1.4 Kernel / Filter 卷积核

卷积核是一个小矩阵。

例如：

```text
3 x 3
5 x 5
```

它在图像上滑动，对局部区域做加权计算。

【核心】卷积核是 CNN 中可学习的局部特征检测器。

### 核心讲解（课程模式 · 0.1.4 · 2026-05-14）

把 **0.1.3** 的「运算」具体到 **一样东西**：卷积核就是 **一小块与 patch 对齐的权重张量**。单通道情形下你把它看成 **`K×K` 实数矩阵**即可；训练中通过反向传播改动这些数，使某项损失下降。

本节再记三件具体事：

```text
1) 一个核在整个空间上重复使用 → 「权重共享」，这是 CNN 参数远少于全连接的关键之一。
2) 一层里通常不止一个核：第 i 个核负责一种（可学习的）局部响应模式，输出第 i 张特征图。
3) 深层的「核」读的不一定是 RGB，而是上一层的特征图拼成的多通道体；那是 0.1.5 与工程里的 in_channels / out_channels。
```

**与上节脚本关系**：`cnn_valid_conv_numpy_demo.py` 里写死的 **`3×3`** 是一段 **常量核**；等价于未经训练的「手写局部检测器」，用来对上 **乘法—累加** 有体感。真实网络里的核 **`requires_grad`**，由 **`loss.backward()`**（第 6 课骨架）迭代更新。

### 拓展阅读

- **下一节 `### 0.1.5 Feature Map`**：从「多张输出」讲到 **低级 / 高级**语义直觉。  
- **反向传播直觉**：某项损失对输出的敏感会一路乘链式法则传到每个核权重上——把核想成「可旋钮的局部模版」足够。

### 联想扩展

- **Java 类比**：若把 **单层卷积**想成：`for each 输出坐标` → `weightedSum(patch, fixedKernelWeights)`——**不同之处**是全图共用 **同一份 Kernel 对象的权重向量**，梯度对 **所有出现位置**的贡献会 **相加**后再更新（实现里已向量化完成）。

### 0.1.5 Feature Map 特征图

卷积层输出的结果叫特征图。

低层特征图可能表示：

- 边缘。
- 颜色变化。
- 纹理。

高层特征图可能表示：

- 眼睛。
- 轮廓。
- 物体部件。

### 核心讲解（课程模式 · 0.1.5 · 2026-05-14）

**特征图（feature map）**在工程里指：**某一层卷积（或类似局部线性算子）在空间上「扫完一遍」之后得到的那张二维响应场**。输入若是 RGB，第一层读的是 **`3×H×W`**；该层若有 **`C_out`** 个核并行扫图，就得到 **`C_out` 张**同尺寸的（或按 0.1.3 公式缩放后的）特征图，其中 **深度维等于 `C_out`**，语义是「并行检测几种局部模式」。

本节记四件事：

```text
1) 张量形状口语：常记 ``(N, C, H, W)``——本课可先只盯单样本时的 ``(C, H, W)``。
2) 第 i 张特征图 ≈ **第 i 个卷积核**在整幅输入上滑出来的「激活图」。
3) 「低层像边缘、高层像部件」是 **经验直觉**：帮助建立层次化表征的心智模型，不是严格可证的分类定义。
4) 深网络里常见 **通道变多、单张图空间略缩小**（靠 stride / pooling 等），像 **从细网格的局部响应 → 更粗网格的语义组合**。
```

**自测**：若 ``C_in=64``、`C_out=128`，且空间高宽未被改变，输出特征图的 **通道维**应是几？——**128**（与核个数一致）。

### 拓展阅读

- **下一小节 `### 0.1.6`**：在 **不下调参数量**（通常无权重）的前提下 **压空间分辨率**。  
- **ResNet / 现代 CNN**：有的块用 **stride>1 的卷积**代替部分池化角色；先把 **池化的直观**抓住即可。

### 联想扩展

- **与 0.1.4 勾连**：**多核**是 **多张特征图**的直接原因；不要再把「RGB 三通道」与「第 40 通道」混成同一语义——前者是输入约定，后者是 **学出来的抽象特征索引**。  
- **Java 类比**：可把 **固定 H×W 上的一层 C 张特征图**想成 **C 个并行的二维 `double[][]` 面板**，面板之间通过下一层核的 **`in_channels`** 再耦合。

### 0.1.6 Pooling 池化

池化用于压缩特征图大小。

常见：

- Max Pooling
- Average Pooling

作用：

- 降低计算量。
- 提高对小范围位置变化的鲁棒性。
- 保留重要特征。

### 核心讲解（课程模式 · 0.1.6 · 2026-05-14）

**池化（pooling）**在局部窗口内做 **固定的汇聚规则**（常见 **Max** 或 **Average**），一般 **不学参数**。它吃一张（或多通道上的同位置窗口）特征响应，吐一个标量，因此 **空间分辨率往往下降**（例如 **`2×2` 窗口、`stride=2`** → 高宽各约减半）。

对比卷积：

```text
- 卷积：窗口内与可学习核做加权和 —— 参数多、表达细。
- 池化：窗口内取 max 或 mean —— 常无参数、偏「抽象与抑噪」。
```

**Max Pooling** 直觉：**这一小块里谁最「响」就信谁**，对微小平移更钝感；**Average Pooling** 直觉：**把局部能量平均掉**，更平滑、有时少丢背景但锐度弱。许多经典视觉网络里 **Max 更常见**，但具体任务会换设计。

```bash
python ai-learning/07-cnn-rnn-transformer-intro/scripts/pooling_2x2_stride2_numpy_demo.py
```

**自测**：输入 **`4×4`**、**`2×2` 池化**、**`stride=2`**、**`padding=0`**，输出空间边长是几？——**2**。

### 0.1.7 RNN

RNN 是 Recurrent Neural Network，循环神经网络。

它用于处理序列数据。

序列数据特点：

```text
当前元素的含义和前后顺序有关。
```

例如：

- 文本。
- 时间序列。
- 语音。
- 用户行为序列。

【核心】RNN 通过隐藏状态保存历史信息。

### 0.1.8 Hidden State 隐藏状态

隐藏状态是 RNN 记忆历史信息的向量。

每一步输入后，RNN 更新隐藏状态：

```text
当前输入 + 上一步隐藏状态 -> 新隐藏状态
```

【重点】隐藏状态是 RNN 处理序列的核心。

### 0.1.9 LSTM / GRU

普通 RNN 容易遇到长距离依赖问题。

LSTM 和 GRU 是改进结构，用门控机制控制信息保留和遗忘。

适合：

- 较长文本。
- 时间序列。
- 需要记忆历史上下文的任务。

【重点】LSTM/GRU 曾是 NLP 主流结构，但大模型时代主要由 Transformer 接替。

### 0.1.10 Transformer

Transformer 是现代大模型的核心结构。

它不依赖循环逐步处理序列，而是通过 Attention 机制直接建模序列中任意位置之间的关系。

【核心】Transformer 是大模型的基础架构。

### 0.1.11 Attention 注意力机制

Attention 的直觉是：

```text
处理当前位置时，模型应该关注序列中哪些其他位置？
```

例如句子：

```text
小明把书放进书包，因为它很重。
```

模型需要判断“它”指什么。

Attention 可以让模型在处理“它”时关注“书”。

### 0.1.12 Self-Attention 自注意力

Self-Attention 是序列内部各位置之间互相关注。

每个 token 都可以看其他 token，并计算相关性。

【核心】Self-Attention 让 Transformer 能建模长距离依赖。

### 0.1.13 Token

Token 是文本被模型处理的基本单位。

它可能是：

- 一个词。
- 一个子词。
- 一个字符片段。
- 一个标点。

【重点】大模型不是直接处理原始字符串，而是先把文本切成 token。

### 0.1.14 Embedding

Embedding 把 token 转成向量。

Transformer 处理的是向量序列，不是原始文本。

```text
文本 -> token -> embedding -> Transformer
```

### 0.1.15 Positional Encoding 位置编码

Transformer 本身没有 RNN 的顺序处理机制，因此需要位置编码告诉模型 token 顺序。

【重点】没有位置信息，Transformer 很难区分词序。

### 0.1.16 Encoder / Decoder

Transformer 常见结构包括：

- Encoder：理解输入。
- Decoder：生成输出。

模型类型：

| 类型 | 示例任务 |
|---|---|
| Encoder-only | 文本分类、理解 |
| Decoder-only | 文本生成、大语言模型 |
| Encoder-Decoder | 翻译、摘要 |

【核心】GPT 类模型通常是 Decoder-only Transformer。

### 0.1.17 本课重点标注汇总

【核心】CNN 擅长局部空间特征，典型用于图像。

【核心】RNN 擅长序列建模，通过隐藏状态传递历史信息。

【核心】Transformer 通过 Self-Attention 建模序列中任意位置关系，是大模型基础。

【重点】卷积核是可学习的局部特征检测器。

【重点】Attention 解决的是“当前处理位置应该关注哪里”的问题。

【重点】Token 和 Embedding 是文本进入模型前的关键转换。

【易错】Transformer 不是只用于文本，也可扩展到图像、语音、多模态。

## 2. 三类结构对比

| 结构 | 擅长数据 | 核心思想 | 典型应用 |
|---|---|---|---|
| CNN | 图像、局部空间结构 | 卷积提取局部特征 | 图像分类、目标检测 |
| RNN | 序列 | 隐藏状态传递历史 | 文本、语音、时间序列 |
| Transformer | 序列和多模态 | Attention 建模全局关系 | 大模型、翻译、摘要 |

## 3. CNN 详解

CNN 利用图像的局部性：

```text
边缘 -> 纹理 -> 部件 -> 物体
```

网络越深，特征越抽象。

【工程经验】CNN 仍然是很多视觉任务的重要基础，即使现在视觉 Transformer 也很流行。

## 4. RNN 详解

RNN 逐步读取序列：

```text
x1 -> h1
x2 + h1 -> h2
x3 + h2 -> h3
```

问题：

- 训练难。
- 长距离依赖弱。
- 难以并行。

这些问题推动了 Transformer 的流行。

## 5. Transformer 详解

Transformer 用 Self-Attention 让每个 token 直接关注其他 token。

优势：

- 并行能力强。
- 长距离依赖更好。
- 可扩展到大规模数据和参数。

【核心】大语言模型的核心能力建立在 Transformer 可扩展性上。

## 6. 从文本到 Transformer

流程：

```text
原始文本
  -> Tokenization
  -> Token IDs
  -> Embedding
  -> Positional Encoding
  -> Transformer Blocks
  -> 输出概率或表示
```

【重点】模型真正处理的是数字向量，不是原始文字。

## 7. 常见误区

1. 以为 CNN 只能用于图片。它也可用于一维信号。
2. 以为 RNN 已经完全没用。它仍可用于小型序列任务。
3. 以为 Transformer 不需要位置信息。
4. 以为 Attention 就是简单关键词匹配。
5. 以为大模型只靠结构，忽略数据和训练规模。

## 8. 代码辅助示例

### 8.1 CNN 层形状直觉

```python
import torch
from torch import nn

x = torch.randn(8, 3, 32, 32)
conv = nn.Conv2d(in_channels=3, out_channels=16, kernel_size=3)
y = conv(x)

print(y.shape)
```

含义：

```text
8：batch size
3：输入通道
32x32：图像大小
16：输出特征图通道数
```

### 8.2 Transformer 输入直觉

```python
import torch
from torch import nn

embedding = nn.Embedding(num_embeddings=10000, embedding_dim=128)
token_ids = torch.tensor([[1, 23, 456, 78]])
x = embedding(token_ids)

print(x.shape)
```

含义：

```text
token id -> embedding 向量
```
