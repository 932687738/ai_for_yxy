# 第 0 课：AI 全局认知与学习定位

## 1. 本课目标

学完本课后，你应该能够：

1. 说清楚 AI、机器学习、深度学习、大模型之间的关系。
2. 区分训练、推理、微调、RAG 的不同用途。
3. 理解 Java 开发者学习 AI 的合理路线。
4. 画出一个企业级大模型应用的基本架构。
5. 明白为什么多数业务系统优先做“AI 应用开发”，而不是“从零训练大模型”。

## 2. 你要先建立的全局地图

AI 不是一个单点技术，而是一组技术体系。可以先用下面这张文字图理解：

```text
人工智能 AI
|
+-- 机器学习 Machine Learning
|   |
|   +-- 监督学习：回归、分类
|   +-- 无监督学习：聚类、降维
|   +-- 推荐系统、异常检测等
|
+-- 深度学习 Deep Learning
|   |
|   +-- CNN：常用于图像
|   +-- RNN/LSTM：早期常用于序列
|   +-- Transformer：现代大模型核心结构
|
+-- 大模型 LLM / Foundation Model
    |
    +-- 文本生成
    +-- 对话问答
    +-- 代码生成
    +-- 多模态理解
    +-- 工具调用与 Agent
```

一句话理解：

- AI 是大概念。
- 机器学习是让机器从数据中学习规律。
- 深度学习是使用多层神经网络学习复杂规律。
- 大模型是基于海量数据和大规模神经网络训练出的通用模型。
- AI 应用开发是把模型能力接入真实业务系统。

## 3. Java 开发者该怎么定位

你有 Java 基础，所以你的优势不是从零开始研究数学公式，也不是马上训练一个百亿参数模型，而是把 AI 能力工程化落地。

更合理的定位是：

```text
Java 业务系统开发能力
        +
AI 基础认知
        +
大模型 API / RAG / Agent
        +
Spring Boot 工程化能力
        =
AI 应用开发者
```

企业里常见的 AI 应用包括：

- 企业知识库问答
- 智能客服
- 合同 / 标书 / 简历解析
- 数据分析助手
- 代码助手
- 业务流程自动化 Agent
- 文档摘要与内容生成
- 质检、分类、推荐、风控辅助系统

这些系统大多不是“自己训练一个大模型”，而是：

1. 调用已有模型。
2. 接入企业内部数据。
3. 做权限、检索、审计、日志、成本控制。
4. 把 AI 能力嵌入现有业务流程。

## 4. 训练、推理、微调、RAG 的区别

这是 AI 应用开发必须先搞清楚的四个词。

### 4.1 训练 Training

训练是让模型从大量数据中学习规律。

```text
大量训练数据 + 模型结构 + 损失函数 + 优化算法 -> 模型参数
```

特点：

- 成本高。
- 需要大量数据和算力。
- 更偏算法工程师、研究员、模型工程师。
- Java 应用开发者通常不从这里起步。

### 4.2 推理 Inference

推理是使用已经训练好的模型完成任务。

```text
用户输入 -> 已训练模型 -> 模型输出
```

例如：

- 输入：“总结这段合同”
- 输出：“该合同主要约定了付款方式、交付周期和违约责任……”

大多数 AI 应用开发，本质上都是围绕推理做工程封装。

### 4.3 微调 Fine-tuning

微调是在已有模型基础上，用特定数据继续训练，让模型更适合某类任务。

适合场景：

- 固定格式输出
- 垂直行业风格适配
- 特定分类任务
- 专业术语表达习惯

不适合一上来就做的情况：

- 只是想让模型回答企业文档内容
- 文档经常变化
- 数据量不大
- 还没有成熟的评估集

这类情况通常优先使用 RAG。

### 4.4 RAG

RAG 是 Retrieval-Augmented Generation，即检索增强生成。

核心思想：

```text
用户问题
  |
  v
检索企业知识库
  |
  v
取回相关资料
  |
  v
把资料和问题一起发给大模型
  |
  v
生成基于资料的回答
```

RAG 的价值：

- 不需要重新训练大模型。
- 可以接入企业私有知识。
- 文档更新后可重新入库。
- 更适合知识库问答、制度问答、产品手册问答等业务。

## 5. 典型大模型应用架构

一个企业知识库问答系统通常长这样：

```text
前端页面 / 企业微信 / 钉钉 / App
        |
        v
Java Spring Boot 后端
        |
        +-- 用户认证与权限
        +-- 会话管理
        +-- Prompt 组装
        +-- RAG 检索
        +-- 模型调用
        +-- 日志审计
        +-- 成本统计
        |
        v
向量数据库 PostgreSQL + pgvector / Milvus / Elasticsearch
        |
        v
企业文档：PDF / Word / Markdown / 数据库记录
        |
        v
大模型 API / 本地模型服务
```

这张图说明了一件事：AI 应用不是只有“调用模型”这一行代码。真正上线时，权限、日志、检索质量、回答质量、成本、延迟、稳定性都很重要。

## 6. 你接下来的学习主线

后续课程会按以下顺序推进：

```text
AI 全局认知
  -> Python 与数学基础
  -> 机器学习
  -> 深度学习
  -> 大模型应用开发
  -> Java AI 工程化
  -> 综合项目
```

你不需要在第一阶段把所有数学都学到研究生水平。对 Java 开发者来说，前期目标是：

- 能看懂 AI 示例代码。
- 能理解模型训练的基本流程。
- 能知道指标、数据、参数在干什么。
- 能把 AI 能力接入业务系统。
