## 0.1 概念讲义：AI 数学术语与核心知识详解

### 0.1.1 第 2 课到底在学什么

第 2 课的任务不是让你成为数学专家，而是让你看懂后续机器学习和深度学习中反复出现的数学语言。

机器学习中经常出现：

```text
X
y
w
b
loss
gradient
learning_rate
embedding

```

这些符号背后对应的就是：

- 向量。
- 矩阵。
- 概率。
- 统计。
- 导数。
- 梯度。
- 损失函数。

【重点】本课的目标是建立“数学概念和 AI 场景之间的映射关系”。

### 0.1.2 标量 Scalar

标量就是一个单独的数。

例如：

```text
价格 = 120
概率 = 0.82
学习率 = 0.01
损失值 = 3.5
```

在 AI 中，标量常用于表示：

- 一个特征值。
- 一个概率。
- 一个损失值。
- 一个参数。
- 一个超参数。

【重点】标量是最小的数值单位。

### 0.1.3 向量 Vector

向量是一组有顺序的数。

一个房子可以表示为：

```text
[面积, 房间数, 房龄]
```

一个用户可以表示为：

```text
[最近登录天数, 订单数, 累计消费金额]
```

一段文本也可以通过 Embedding 表示为：

```text
[0.12, -0.33, 0.85, ...]
```

【核心】向量是 AI 中表示一个对象的常见方式。

你可以把向量理解为：

```text
把一个现实对象拆成多个数字维度后的表示。
```

### 0.1.4 维度 Dimension

维度表示向量里有多少个数字。

例如：

```text
[100, 3, 8]
```

是 3 维向量。

Embedding 可能是：

```text
1536 维向量
3072 维向量
```

【重点】维度越高，能表达的信息可能越丰富，但计算和理解也更复杂。

【易错】维度高不等于一定更好，关键看表示是否有效。

### 0.1.5 矩阵 Matrix

矩阵是二维数字表格。

在机器学习中，特征矩阵通常长这样：

```text
每一行：一个样本
每一列：一个特征
```

例如：

```text
[
  [80, 2, 10],
  [100, 3, 8],
  [120, 3, 5]
]
```

表示 3 个房子，每个房子 3 个特征。

【核心】矩阵是批量样本的数字表示。

### 0.1.6 张量 Tensor

张量可以理解为多维数组。

```text
0 维：标量
1 维：向量
2 维：矩阵
3 维及以上：张量
```

图片数据常用张量表示：

```text
高度 x 宽度 x 颜色通道
```

深度学习框架 PyTorch 中最核心的数据对象就叫 Tensor。

【重点】深度学习中的数据、参数、中间计算结果，通常都以张量形式存在。

### 0.1.7 shape

shape 表示数组、矩阵、张量的形状。

例如：

```text
X.shape = (100, 5)
```

通常表示：

```text
100 个样本
5 个特征
```

【核心】shape 是调试 AI 代码最重要的信息之一。

【易错】很多 AI 代码报错不是算法错，而是 shape 不匹配。

### 0.1.8 点积 Dot Product

点积是两个向量对应元素相乘后求和。

例如：

```text
[1, 2, 3] · [10, 20, 30]
= 1*10 + 2*20 + 3*30
= 140
```

在 AI 中，点积常用于：

- 线性模型预测。
- 神经网络加权求和。
- 计算向量相关性。
- Attention 机制中的相似度计算。

【核心】点积是“特征值和权重共同作用”的数学表达。

### 0.1.9 权重 Weight

权重表示某个特征对预测结果的影响强度。

例如：

```text
price = 1.5 * area + 10 * rooms - 2 * age
```

其中：

```text
1.5、10、-2
```

都是权重。

【重点】模型训练的一个重要目标，就是学习合适的权重。

### 0.1.10 偏置 Bias

偏置是模型中的常数项。

例如：

```text
price = 1.5 * area + 10 * rooms - 2 * age + 5
```

这里的 `5` 就是偏置。

它可以理解为模型的基础起点。

【重点】线性模型通常由权重和偏置共同决定预测结果。

### 0.1.11 范数 Norm

范数可以理解为向量的长度。

例如二维向量：

```text
[3, 4]
```

长度是：

```text
5
```

因为：

```text
sqrt(3^2 + 4^2) = 5
```

在 AI 中，范数常用于：

- 计算距离。
- 归一化向量。
- 正则化模型参数。
- 计算余弦相似度。

### 0.1.12 欧氏距离 Euclidean Distance

欧氏距离衡量两个点之间的直线距离。

它回答：

```text
两个向量在空间中离得有多远？
```

在推荐、聚类、检索中，距离常用于衡量样本相似程度。

【重点】距离小通常表示更接近，但在高维语义向量中，余弦相似度更常见。

### 0.1.13 余弦相似度 Cosine Similarity

余弦相似度衡量两个向量方向是否接近。

它不重点关注长度，而关注方向。

在 Embedding 检索中：

```text
问题向量和文档向量方向越接近，语义越相似。
```

【核心】RAG 和语义搜索的基础之一就是向量相似度。

【易错】余弦相似度不是判断文本字面是否相同，而是判断向量表示的语义是否接近。

### 0.1.14 Embedding

Embedding 是把文本、图片、商品、用户等对象转换成向量表示。

例如：

```text
"Java AI 应用开发" -> [0.12, -0.08, 0.91, ...]
```

Embedding 的价值是：

```text
把非结构化对象变成可以计算相似度的数字向量。
```

【核心】Embedding 是大模型应用、语义搜索、RAG 的基础概念。

### 0.1.15 概率 Probability

概率描述不确定性。

例如：

```text
用户流失概率 = 0.82
邮件是垃圾邮件概率 = 0.97
图片是猫的概率 = 0.91
```

概率不是绝对事实，而是模型在现有信息下的判断。

【重点】模型输出概率后，业务系统通常还需要根据阈值做决策。

### 0.1.16 条件概率 Conditional Probability

条件概率表示在某个条件发生的情况下，另一个事件发生的概率。

例如：

```text
P(用户流失 | 最近 30 天未登录)
```

意思是：

```text
已知用户最近 30 天未登录，该用户流失的概率。
```

【重点】很多机器学习预测可以理解为在给定特征条件下估计目标发生的概率。

### 0.1.17 随机变量 Random Variable

随机变量是结果不确定的变量。

例如：

```text
明天某商品销量
某用户是否流失
一次点击是否转化
```

这些值在发生前是不确定的，只能用概率描述。

### 0.1.18 均值 Mean

均值表示一组数据的平均水平。

例如：

```text
平均房价
平均订单金额
平均登录次数
```

【重点】均值能描述整体水平，但容易受极端值影响。

### 0.1.19 方差 Variance 和标准差 Standard Deviation

方差和标准差描述数据波动程度。

如果标准差大，说明数据分散。

如果标准差小，说明数据集中。

在 AI 中，它们用于：

- 判断数据是否稳定。
- 发现异常值。
- 理解特征分布。
- 做标准化处理。

### 0.1.20 分布 Distribution

分布描述数据出现的整体形态。

例如：

```text
房价集中在哪个区间？
大部分用户消费金额是多少？
是否少数用户消费特别高？
类别是否严重不平衡？
```

【核心】训练模型前必须先观察数据分布。

### 0.1.21 导数 Derivative

导数描述变化率。

它回答：

```text
当 x 变化一点点时，y 会变化多少？
```

在模型训练中，我们关心：

```text
参数变化一点点，loss 会变大还是变小？
```

【核心】导数告诉我们参数调整的方向信息。

### 0.1.22 梯度 Gradient

梯度是多个参数对应导数组成的向量。

如果模型有多个参数：

```text
w1, w2, w3, b
```

那么梯度可能是：

```text
[dw1, dw2, dw3, db]
```

【核心】梯度告诉模型每个参数应该如何调整，才能让损失下降。

### 0.1.23 损失函数 Loss Function

损失函数衡量模型预测有多错。

例如：

```text
真实房价 = 150
预测房价 = 130
误差 = -20
```

损失函数会把误差转换成一个可优化的数值。

【核心】模型训练的目标就是让损失函数尽可能小。

### 0.1.24 MSE

MSE 是均方误差。

它先把误差平方，再求平均。

适合回归任务。

【重点】平方会放大大误差，所以 MSE 对大错误更敏感。

### 0.1.25 学习率 Learning Rate

学习率控制每次参数更新的步长。

学习率太小：

```text
训练很慢。
```

学习率太大：

```text
可能震荡或发散。
```

【核心】学习率是训练模型时最重要的超参数之一。

### 0.1.26 梯度下降 Gradient Descent

梯度下降是一种优化方法。

它的目标是：

```text
沿着让 loss 下降的方向，一步步更新参数。
```

参数更新公式可以理解为：

```text
新参数 = 旧参数 - 学习率 * 梯度
```

【核心】梯度下降是理解机器学习训练过程的基础。

【核心】梯度下降通过不断更新参数来降低损失。

## 4. 标量、向量、矩阵、张量

### 4.1 标量 Scalar

标量就是一个单独的数。

例如：

```text
年龄 = 30
价格 = 99.5
学习率 = 0.01
损失值 = 2.35
```

Python 示例：

```python
age = 30
learning_rate = 0.01
loss = 2.35
```

在 AI 中，标量常用于：

- 一个特征值
- 一个模型参数
- 一个损失值
- 一个概率值
- 一个学习率

### 4.2 向量 Vector

向量是一组有顺序的数。

例如，一个房子可以用 3 个特征表示：

```text
面积 = 100
房间数 = 3
房龄 = 8
```

可以写成向量：

```text
[100, 3, 8]
```

Python / NumPy 表示：

```python
import numpy as np

house = np.array([100, 3, 8])
print(house)
print(house.shape)
```

输出：

```text
[100   3   8]
(3,)
```

在机器学习中，一个样本通常就是一个向量。

```text
一个用户 = [年龄, 消费次数, 最近登录天数, 是否会员]
一个商品 = [价格, 类目编号, 销量, 评分]
一段文本 = [0.12, -0.33, 0.85, ...]  # Embedding 向量
```

### 4.3 矩阵 Matrix

矩阵是二维数字表格。

如果有 5 套房子，每套房子 3 个特征：

```text
[80, 2, 10]
[100, 3, 8]
[120, 3, 5]
[150, 4, 3]
[60, 1, 15]
```

可以组成矩阵：

```python
import numpy as np

houses = np.array([
    [80, 2, 10],
    [100, 3, 8],
    [120, 3, 5],
    [150, 4, 3],
    [60, 1, 15],
])

print(houses.shape)
```

输出：

```text
(5, 3)
```

含义：

```text
5 行：5 个样本
3 列：3 个特征
```

机器学习中常用记号：

```text
X = 输入特征矩阵
y = 目标值
```

例如：

```text
X.shape = (样本数量, 特征数量)
y.shape = (样本数量,)
```

### 4.4 张量 Tensor

张量可以简单理解为多维数组。

```text
0 维张量：标量
1 维张量：向量
2 维张量：矩阵
3 维及以上：更高维数组
```

图片可以看成张量：

```text
灰度图片：高度 x 宽度
彩色图片：高度 x 宽度 x 颜色通道
一批彩色图片：批次大小 x 高度 x 宽度 x 颜色通道
```

深度学习框架 PyTorch 里最核心的数据结构就叫 Tensor。

## 5. shape：AI 代码里最重要的调试信息

AI 初学者写代码最常见的错误之一是维度不匹配。

例如矩阵乘法要求：

```text
(m, n) @ (n, p) = (m, p)
```

中间的 `n` 必须相同。

示例：

```python
import numpy as np

X = np.array([
    [80, 2, 10],
    [100, 3, 8],
])

w = np.array([1.5, 10, -2])

print(X.shape)
print(w.shape)

y_pred = X @ w
print(y_pred)
print(y_pred.shape)
```

输出：

```text
(2, 3)
(3,)
[120. 164.]
(2,)
```

解释：

```text
X 有 2 个样本，每个样本 3 个特征
w 有 3 个权重
X @ w 会得到每个样本的预测值
```

调试 AI 代码时，建议频繁打印：

```python
print(X.shape)
print(y.shape)
print(weights.shape)
```

## 6. 向量运算

### 6.1 向量加法


### 6.3 点积 Dot Product

点积是两个向量对应元素相乘后求和。

```python
import numpy as np

a = np.array([1, 2, 3])
b = np.array([10, 20, 30])

dot = np.dot(a, b)
print(dot)
```

计算过程：

```text
1*10 + 2*20 + 3*30 = 140
```

点积在 AI 中非常重要。

典型场景：

1. 线性模型预测。
2. 神经网络加权求和。
3. 向量相似度计算。
4. Attention 机制中的相关性计算。

线性模型可以写成：

```text
y = x1*w1 + x2*w2 + x3*w3 + b
```

用向量写就是：

```text

```


计算过程：

```text
100*1.5 + 3*10 + 8*(-2) + 5 = 169
```

这就是一个极简房价预测公式。

## 7. 向量长度、距离与相似度

### 7.1 向量长度 Norm

二维向量 `[3, 4]` 的长度是 5，因为：

```text
sqrt(3^2 + 4^2) = 5
```

NumPy 示例：

```python
import numpy as np

v = np.array([3, 4])
length = np.linalg.norm(v)
print(length)
```

输出：

```text
5.0
```

更一般地，向量 `[x1, x2, ..., xn]` 的长度是：

```text
sqrt(x1^2 + x2^2 + ... + xn^2)
```

### 7.2 欧氏距离

欧氏距离衡量两个点之间的直线距离。

```python
import numpy as np

a = np.array([1, 2])
b = np.array([4, 6])

distance = np.linalg.norm(a - b)
print(distance)
```

计算：

```text
a - b = [-3, -4]
sqrt((-3)^2 + (-4)^2) = 5
```

### 7.3 余弦相似度

余弦相似度衡量两个向量方向是否接近。

公式：

```text
cosine_similarity(a, b) = (a · b) / (||a|| * ||b||)
```

Python 实现：

```python
import numpy as np


def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))


query = np.array([1.0, 0.9, 0.1])
doc_a = np.array([0.9, 0.8, 0.2])
doc_b = np.array([0.1, 0.2, 0.9])

print(cosine_similarity(query, doc_a))
print(cosine_similarity(query, doc_b))
```

`doc_a` 和 `query` 更相似，因为它们方向更接近。

这就是向量检索和 RAG 的基础思想：

```text
用户问题 -> Embedding 向量
文档片段 -> Embedding 向量
计算问题向量和文档向量的相似度
返回最相似的文档片段
```

## 10. 概率基础

### 10.1 概率是什么

概率用于描述不确定性。

例如：

```text
明天下雨的概率是 70%
用户流失的概率是 20%
这张图片是猫的概率是 95%
这封邮件是垃圾邮件的概率是 98%
```

概率值范围：

```text
0 <= P <= 1
```

含义：

```text
0：不可能发生
1：一定发生
0.5：一半可能
```

### 10.2 分类模型输出为什么是概率

一个二分类模型可能输出：

```text
用户流失概率 = 0.82
```

业务系统可以据此决策：

```text
if probability >= 0.8:
    标记为高流失风险用户
else:
    正常跟进
```

多分类模型可能输出：

```text
猫：0.80
狗：0.15
兔子：0.05
```

概率最高的类别通常作为预测结果。

### 10.3 条件概率

条件概率表示“在某个条件已经发生的情况下，另一个事件发生的概率”。

记作：

```text
P(A | B)
```

读作：

```text
在 B 发生的条件下，A 发生的概率
```

业务例子：

```text
P(用户流失 | 最近 30 天未登录)
```

这表示：

```text
已知用户最近 30 天未登录的情况下，该用户流失的概率
```

机器学习里很多判断都可以理解为条件概率估计。

### 10.4 期望、方差、标准差

假设一组分数：

```python
import numpy as np

scores = np.array([60, 70, 80, 90, 100])

print(np.mean(scores))
print(np.var(scores))
print(np.std(scores))
```

含义：

| 指标 | 含义 |
|---|---|
| 均值 mean | 平均水平 |
| 方差 variance | 数据波动程度 |
| 标准差 standard deviation | 数据离均值的典型距离 |

如果两个班平均分都是 80：

```text
A 班：78, 79, 80, 81, 82
B 班：50, 70, 80, 90, 110
```

平均分相同，但 B 班波动更大，标准差更大。

AI 数据分析中，标准差常用于判断：

- 数据是否分散
- 是否存在异常值
- 特征是否需要标准化

## 11. 数据分布

### 11.1 为什么要看分布

训练模型前，不要急着写训练代码。先看数据分布。

例如房价数据：

```python
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv("data/houses.csv")

plt.hist(df["price"], bins=5)
plt.xlabel("Price")
plt.ylabel("Count")
plt.title("Price Distribution")
plt.show()
```

你需要观察：

1. 数据是否集中在某个范围。
2. 是否有极端异常值。
3. 是否严重偏斜。
4. 不同类别样本是否均衡。

### 11.2 正态分布直觉

正态分布常被称为钟形曲线。

```text
          *
        *   *
      *       *
    *           *
  *               *
*                   *
```

很多自然现象近似正态分布，例如身高、测量误差等。

但真实业务数据不一定正态。例如：

- 用户消费金额可能长尾分布。
- 商品销量可能少数商品很高，大量商品很低。
- 日志访问量可能有明显峰值。

模型训练前了解数据分布，可以帮助你选择合适的处理方式。

## 12. 导数与变化率

### 12.1 导数的直觉

导数描述的是“变化率”。

如果函数是：

```text
y = x^2
```

当 x 变化一点点时，y 会变化多少？

这就是导数关心的问题。

例如：

```text
y = x^2
y' = 2x
```

当：

```text
x = 3
```

导数：

```text
y' = 2 * 3 = 6
```

含义：

```text
在 x = 3 附近，x 每增加一点，y 大约以 6 倍速度增加
```

### 12.2 为什么 AI 需要导数

模型训练要解决的问题是：

```text
怎样调整参数，才能让预测误差更小？
```

如果损失函数是：

```text
loss = (prediction - target)^2
```

我们需要知道：

```text
参数 w 增大一点，loss 会变大还是变小？
参数 b 增大一点，loss 会变大还是变小？
```

导数告诉我们变化方向和变化速度。

### 12.3 梯度

如果只有一个参数，导数是一个数。

如果有多个参数，每个参数都有一个导数，这一组导数组成梯度。

```text
参数：w1, w2, w3, b
梯度：[dw1, dw2, dw3, db]
```

梯度告诉模型：

```text
每个参数应该往哪个方向调整，loss 才会下降
```

## 13. 损失函数

损失函数用于衡量模型预测结果有多差。

例如真实房价是 150，模型预测是 130：

```text
误差 = 130 - 150 = -20
```

平方误差：

```text
(-20)^2 = 400
```

多个样本时，常用均方误差 MSE：

```text
MSE = 所有平方误差的平均值
```

Python 示例：

```python
import numpy as np

y_true = np.array([120, 150, 180])
y_pred = np.array([110, 160, 200])

mse = np.mean((y_pred - y_true) ** 2)
print(mse)
```

损失函数越小，说明模型预测越接近真实值。

训练模型的核心目标：

```text
找到一组参数，让损失函数尽可能小
```

## 14. 梯度下降

### 14.1 核心思想

梯度下降是一种寻找较小损失值的方法。

直觉：

```text
你站在山坡上
想走到山谷最低点
每一步都朝最陡的下降方向走一点
```

在模型训练中：

```text
山坡高度 = loss
当前位置 = 当前模型参数
下降方向 = 梯度的反方向
步长 = 学习率 learning rate
```

参数更新公式：

```text
new_parameter = old_parameter - learning_rate * gradient
```

### 14.2 学习率

学习率控制每次更新走多大一步。

```text
学习率太小：训练很慢
学习率太大：可能越过最低点，甚至发散
```

示意：

```text
太小：
当前位置 -> . -> . -> . -> 最低点

合适：
当前位置 -> -> -> 最低点

太大：
当前位置 -> 越过最低点 -> 来回震荡
```

## 15. 从零实现线性回归训练

现在用一个极简例子，把前面的概念串起来。

我们希望模型学会：

```text
y = 2x + 1
```

但模型一开始不知道 2 和 1，它只有随机参数：

```text
y_pred = w*x + b
```

目标是通过训练，让：

```text
w 接近 2
b 接近 1
```

### 15.1 完整代码

创建文件：

```text
src/linear_regression_from_scratch.py
```

写入：

```python
import numpy as np


def main() -> None:
    # 训练数据：真实规律是 y = 2x + 1
    X = np.array([1, 2, 3, 4, 5], dtype=float)
    y = np.array([3, 5, 7, 9, 11], dtype=float)

    # 初始化参数
    w = 0.0
    b = 0.0

    learning_rate = 0.01
    epochs = 1000
    n = len(X)

    for epoch in range(epochs):
        # 1. 前向计算：根据当前参数做预测
        y_pred = w * X + b

        # 2. 计算损失：均方误差
        loss = np.mean((y_pred - y) ** 2)

        # 3. 计算梯度
        dw = (2 / n) * np.sum((y_pred - y) * X)
        db = (2 / n) * np.sum(y_pred - y)

        # 4. 更新参数
        w = w - learning_rate * dw
        b = b - learning_rate * db

        if epoch % 100 == 0:
            print(f"epoch={epoch}, loss={loss:.6f}, w={w:.4f}, b={b:.4f}")

    print("Final model:")
    print(f"y = {w:.4f} * x + {b:.4f}")


if __name__ == "__main__":
    main()
```

运行：

```powershell
python src/linear_regression_from_scratch.py
```

你会看到 loss 逐步变小，w 接近 2，b 接近 1。

### 15.2 训练循环拆解

机器学习训练通常都有类似结构：

```text
for epoch in range(epochs):
    预测
    计算损失
    计算梯度
    更新参数
```

这也是后续 PyTorch 训练代码的核心结构：

```python
for epoch in range(epochs):
    y_pred = model(X)
    loss = loss_fn(y_pred, y)
    optimizer.zero_grad()
    loss.backward()
    optimizer.step()
```

区别是：

- NumPy 示例里，我们手写梯度。
- PyTorch 中，框架自动计算梯度。

但思想完全一致。

## 16. Embedding 相似度小实验

这里先不用真实大模型生成 Embedding，手动构造几个向量模拟语义。

创建文件：

```text
src/embedding_similarity_demo.py
```

写入：

```python
import numpy as np


def cosine_similarity(a: np.ndarray, b: np.ndarray) -> float:
    return float(np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b)))


def main() -> None:
    query = np.array([0.9, 0.8, 0.1])

    documents = {
        "Java Spring Boot 开发": np.array([0.88, 0.82, 0.12]),
        "机器学习模型训练": np.array([0.75, 0.70, 0.25]),
        "周末旅游攻略": np.array([0.10, 0.20, 0.95]),
    }

    scores = []
    for title, vector in documents.items():
        score = cosine_similarity(query, vector)
        scores.append((title, score))

    scores.sort(key=lambda item: item[1], reverse=True)

    for title, score in scores:
        print(f"{title}: {score:.4f}")


if __name__ == "__main__":
    main()
```

运行后你会看到和 query 方向最接近的文档排在前面。

这就是后续 RAG 检索的数学基础。

### 先讲再练：三个最常卡点（本轮必讲）

下面三个概念，对应教材前文；这里用**考试/写代码时一定会遇到**的说法收口一遍。讲清楚之后，再跑脚本、`shape` 才对得上号。

#### 1）`X.shape = (100, 5)` 是什么意思？

在**本课程默认约定**下（也是 NumPy /  sklearn 最常见约定）：

- **`100`**：样本个数，即有 **100 行**，每一行是一个观测（一个人、一笔订单、一条文本的一条特征记录……）。
- **`5`**：每个样本有 **5 个特征**，即有 **5 列**，每一列是一种输入（年龄、面积、点击率等）。

记忆口诀：**`shape` 里第一个数多半是「多少条数据」，第二个数是「每条数据多少维特征」**。  
（若遇到三维张量，再往后面加维度；本课先把二维矩阵吃透。）

#### 2）余弦相似度大，说明什么？

余弦相似度比较的是两个向量**方向是否相近**，**不怎么关心长度**（和欧氏距离不同）。

- 两向量**方向越接近**，余弦值越接近 **1**，常解释为「在特征空间里更相似」。
- 方向垂直时接近 **0**；方向相反时接近 **-1**。

在 **RAG / 语义检索** 里：查询向量与文档向量余弦高，通常表示模型认为「语义上更相关」（具体还取决于 embedding 质量与训练数据）。

#### 3）梯度下降里 `learning_rate`（学习率）太大？

`learning_rate` 是**每次沿梯度更新参数时「迈多大一步」**。

- **太大**：步子迈过头，可能在最优解附近**来回震荡**，loss **不降反升**，甚至**发散**（数值爆炸）。
- **太小**：收敛慢，训练费时间，但一般更稳。

实践上：先从较小学习率试起，再按验证集/loss 曲线调参。

## 24. 第 2 课全文串讲（按教材顺序 · 读完再练）

> 下面按教材 **§0 → §17** 顺序一次性串起全课逻辑；细节与公式仍以正文为准，此处负责「为什么、怎么串、写代码时怎么用」。

### §0 ～ §2：怎么学、心态

- **目标**：建立「数学符号 ↔ AI 场景」的映射，能读懂 `X, y, w, loss, gradient`，而不是背推导。
- **心态**：用 Java 的工程严谨性读 Python/NumPy；遇到 `shape` 报错先想维度，不要先猜 API。

### §3：数学地图

- **表示**：标量、向量、矩阵、张量描述数据与参数的规模关系。
- **比较**：范数、距离、相似度描述「有多像」。
- **不确定**：概率、分布描述「有多大可能」。
- **优化**：损失 + 梯度下降描述「怎么把误差变小」。

### §4：标量、向量、矩阵、张量

- **标量**：一个数（损失、学习率、某个特征的一个取值）。
- **向量**：有序数组 → **一个对象**在多个维度上的取值（样本特征向量、embedding）。
- **矩阵**：二维表 → **多行样本、每行一条向量**；或线性变换（看上下文）。
- **张量**：三维及以上（图像 N×H×W×C、批数据 batch×seq×dim）。
- **记忆**：从标量到张量，是「数 → 一条样本 → 一批样本 → 带额外结构的数据」。

### §5：`shape`

- `(n_samples, n_features)`：最常见——**行=样本，列=特征**。
- 调试时先看 `print(x.shape)`，再谈矩阵乘法是否合法（内维对齐）。

### §6：向量运算

- **逐元素**加减：`a + b`（维数需一致或可广播）。
- **数乘**：缩放整条向量。
- **点积** `a·b`：同一维上对应相乘再求和 → 与线性模型、相似度都相关。
- **广播**：小规模代码里常见，先理解「谁被复制到和谁对齐」，否则易 silent bug。

### §7：范数、距离、相似度

- **范数/长度**：向量「有多大」。
- **欧氏距离**：两端点直线距离；embedding 里也常用。
- **余弦相似度**：看**方向**是否接近，长度不敏感 → **语义检索、RAG** 常用。
- **直觉**：「距离」问离多远；「余弦」问指向是否一致。

### §8：矩阵运算

- **矩阵乘法** `A @ B`（或 `np.matmul`）：行乘列累加；**数据维度**与**线性组合**的核心操作。
- 与**逐元素乘** `*` 区分：后者是 Hadamard，维度要求不同、语义完全不同。

### §9：从表格到矩阵

- 业务表一行一条记录 → **`X` 行是样本**；`y` 常是目标列（回归连续值、分类标签等）。
- 这是 **sklearn / PyTorch DataLoader** 心智模型的基础。

### §10：概率基础

- **概率**：0～1 的不确定度量；分类模型输出可解释为各类别概率（校准另说）。
- **条件概率** `P(A|B)`：在 B 成立前提下 A 的可能性 → **很多监督学习本质是学条件分布**。
- **均值/方差/标准差**：数据中心与波动；**标准化特征、判断异常、比较两批数据**都依赖它们。

### §11：数据分布

- **训练前先 EDA**：直方图、是否偏斜、长尾、类别不平衡，决定是否**截断、对数变换、重采样**等。
- **不必正态**：业务数据常偏态；知道分布才能选对损失与预处理。

### §12：导数与梯度

- **导数**：一元函数在某点的变化率 → **参数动一点，loss 怎么变**。
- **梯度**：多元时，各参数偏导组成的向量 → **最陡上升方向**；梯度下降沿 **负梯度** 走以**减小 loss**。

### §13：损失函数

- **损失**：单样本或 batch 上「预测与真实差多少」的度量。
- **MSE**：回归里常用的平方误差平均；可导、惩罚大误差更狠。
- **训练目标**：找参数使**期望损失**（在训练集上的经验近似）最小。

### §14：梯度下降

- **更新**：`θ ← θ - η ∇L`，`η` 即学习率。
- **学习率**：太小慢、太大震荡或发散；与 batch、正则、网络深度联动调节。

### §15：线性回归串起来

- 模型 `ŷ = wx + b`，MSE，对 `w,b` 求导得到梯度，循环更新 → **你后续理解的「反向传播」是此思想的系统版**。
- 与仓库脚本 `ai-learning/02-ai-math-foundation/scripts/linear_regression_from_scratch.py` **一一对应**（读完正文后再跑）。

### §16：Embedding 与相似度

- 高维向量表示语义；用**余弦或点积**做近邻 → **RAG 检索**的数学底色。

### §17：常见误区（务必扫一眼）

- **`shape` 看错维**：矩阵乘法失败或广播误用。
- **混淆 `@` 与 `*`**。
- **不做数据分布就训练**：指标虚高或根本学不动。
- **学习率不调**：loss 不降怪「模型不够大」前先怪优化。

### 回合 2（2026-05-12）：§5～§7 —— `shape`、向量运算、距离与相似度

**§5 `shape`：排错第一站**  
- `shape` 告诉你数组每一维有多长。二维矩阵最常写成 `(行数, 列数)`。  
- 本课约定：**`X.shape = (100, 5)` = 100 个样本、每个样本 5 个特征**（100 行、5 列）。  
- 写矩阵乘法前先想：**左边列数是否等于右边行数**；不对就打印 `a.shape, b.shape`，不要猜。

**§6 向量运算（写好别混）**  
- **加减**：同长度向量逐位置相加（语义：两个特征向量在同样维度上叠加或对比）。  
- **数乘**：向量整体缩放（语义：特征统一放大/缩小，类似调权重直观）。  
- **点积** `a·b`：对应分量相乘再求和。在代码里常写 `np.dot(a, b)`。  
  - 一维时若长度相同，点积是一个**标量**。  
  - **线性模型**里「权重向量 · 特征向量 + 偏置」就是在做点积思想（批处理时会升级成矩阵乘）。  
- **广播（broadcasting）**：长度不一致时 NumPy 可能自动补维对齐；能省事也会**悄悄错**，所以仍要以 `shape` 为准。

**§6 举例（手算 + NumPy 对照）**

1. **逐位置加减（长度必须一致）**  
   - `a = [1, 2, 3]`，`b = [10, 20, 30]`  
   - `a + b = [11, 22, 33]`，`a - b = [-9, -18, -27]`  
   - 语义：同一套特征维度上的两个观测（或「基准 + 偏移」）逐维合并。

2. **数乘**  
   - `2 * [1, 2, 3] = [2, 4, 6]`  
   - 语义：所有特征按同一比例放大（例如统一换算单位、或学习率作用在梯度向量上也是「整体步长」思想）。

3. **点积（标量结果）**  
   - `a = [1, 2, 3]`，`w = [0.5, 1.0, 0.0]`  
   - `a·w = 1*0.5 + 2*1.0 + 3*0.0 = 2.5`  
   - 语义：线性打分——每个特征乘权重再求和（再加偏置就是标量预测 \(\hat y\)）。

4. **广播（省事与踩坑各一例）**  
   - **省事例**：`X` 形状 `(4, 3)`（4 个样本，每样本 3 维特征），`b_row = [1, 0, -1]` 形状 `(3,)`。  
     `X + b_row` 时 NumPy 会把 `b_row` **沿第 0 维复制**，相当于每一行都加同一个向量（给每个样本同一偏移）。结果仍是 `(4, 3)`。  
   - **踩坑例**：`v` 形状 `(3, 1)`（列向量），`w` 形状 `(3,)`。`v + w` 可能按广播规则扩成 `(3, 3)`，和你心里以为的「两个 3 维向量相加」不一致。  
     **对策**：先 `print(v.shape, w.shape)`；需要一维时显式 `v = v.ravel()` 或 `reshape(-1)`。

```python
import numpy as np

a = np.array([1.0, 2.0, 3.0])
b = np.array([10.0, 20.0, 30.0])
print(a + b)           # [11. 22. 33.]
print(2.0 * a)         # [2. 4. 6.]
w = np.array([0.5, 1.0, 0.0])
print(np.dot(a, w))    # 2.5

X = np.array([[1, 0, 2], [0, 1, 1], [2, 2, 0], [1, 1, 1]], float)
bias = np.array([1.0, 0.0, -1.0])
print((X + bias).shape)  # (4, 3)
```

**§6 补：广播（broadcasting）是怎么「算」的**

NumPy **不会真的复制整块大块内存**（常见实现是**按需要对某一维做步长视角或子块循环**），但你可以用下面规则**推断结果形状**和**每个位置上的数怎么来的**。

**规则（从右向左对齐维度）**  
1. 两个数组的 `shape` 在**右侧对齐**（维度少的左边补「虚无的 1」）。  
2. 每一维上，两个长度必须 **相等**，或 **其中一个为 1**，否则报错。  
3. 结果为 **那一维上两者的较大值**（`1` 与 `n` → `n`）。  
4. 在运算时，长度为 `1` 的那一维被**逻辑上**沿该维**重复**到与另一数组相同长度，再**逐元素**做 `+`、`*` 等。

**例 A**：`(4, 3) + (3,)`  
- 对齐成 `(4, 3)` 与 `(1, 3)`（左边给 `(3,)` 补一维）。  
- 右维：3=3；左维：4 与 1 → 取 4，且 `(3,)` 在「第 0 维」上是 1，沿竖方向**每一行**用同一个 `[…]`。  
- 结果 **`(4, 3)`**，第 `i` 行 = `X[i] + bias`。

**例 B**：`(3, 1) + (1, 3)`（经典「十字」扩展）  
- 右维：1 与 3 → 广播成 3；左维：3 与 1 → 广播成 3。  
- 结果 **`(3, 3)`**。  
- 位置 `(i, j)` 上的值 = `A[i,0] + B[0,j]`（列向量复制成多列，行向量复制成多行）。

**例 C**：`(3,) + (3,)` → `(3,)`，无广播，就是逐元素加。

**工程习惯**：对「每个样本同一偏置」用 `(3,)` 或 `(1, 3)` 加到 `(n, 3)` 上；若你本意是**两个样本向量**相加，先统一成 **`(3,)`** 两个一维，不要误用 `(3,1)` 与 `(3,)`。

**§7 长度、距离、相似度（三种常用问法）**  
1. **范数/长度**（常默认 L2）：向量「从原点到端点」有多远。  
2. **欧氏距离**：两个向量终点之间的直线距离，与长度相关：\(\|a-b\|_2\)。问的是「**离多远**」。  
3. **余弦相似度**：看两向量**夹角**，更关心**方向是否一致**，对长度不那么敏感。问的是「**指向是否相近**」（检索、RAG、embedding 近邻常用）。

**对照记忆**  
- 房价两个样本：若更关心「综合差距有多大」→ 可看欧氏距离。  
- 两句文本 embedding：若更关心「语义方向是否像」→ 常看余弦相似度。

**§7 举例（手算小向量）**

设 `u = [3, 4]`，`v = [0, 2]`（可想象二维平面上两个箭头）。

1. **L2 范数（长度）**  
   - \(\|u\|_2 = \sqrt{3^2 + 4^2} = \sqrt{25} = 5\)  
   - \(\|v\|_2 = \sqrt{0^2 + 2^2} = 2\)

2. **欧氏距离**  
   - \(u - v = [3, 2]\)，\(\|u-v\|_2 = \sqrt{3^2 + 2^2} = \sqrt{13} \approx 3.61\)  
   - 含义：两个特征向量在「数值空间」里隔多远。

3. **余弦相似度**（与长度关系弱）  
   - \(\cos(u,v) = \frac{u\cdot v}{\|u\|_2\|v\|_2}\)，其中 \(u\cdot v = 3*0 + 4*2 = 8\)。  
   - \(\cos = 8 / (5 * 2) = 0.8\)  
   - 若把 `u` 整体拉长成 `[30, 40]`（方向不变），点积变大、长度也变大，**余弦仍为 0.8**；欧氏距离会变。故：**比方向用余弦，比绝对位置差用距离**。

```python
import numpy as np
u = np.array([3.0, 4.0])
v = np.array([0.0, 2.0])
print(np.linalg.norm(u))           # 5.0
print(np.linalg.norm(u - v))       # sqrt(13)
print(np.dot(u, v) / (np.linalg.norm(u) * np.linalg.norm(v)))  # 0.8
```

**§7 补：L2 长度、欧氏距离、余弦相似度（详细拆解 `u=[3,4]`, `v=[0,2]`）**

**1）L2 范数（长度）‖·‖₂ 是什么**  
- 把向量看成从**原点 O=(0,0)** 指向**点 (3,4)** 的一条箭头。L2 长度就是这条箭头在平面上的**欧几里得长度**（勾股定理）。  
- 对二维 \((x,y)\)：\(\|u\|_2 = \sqrt{x^2+y^2}\)。  
- **代入 u**：\(x=3,y=4\) → \(\sqrt{9+16}=\sqrt{25}=5\)。即 u 箭头长度为 **5**。  
- **代入 v**：\(x=0,y=2\) → \(\sqrt{0+4}=\sqrt{4}=2\)。v 在纵轴上，从原点到 (0,2)，长度为 **2**。  
- 高维同理：\(\|u\|_2 = \sqrt{u_1^2+u_2^2+\cdots+u_d^2}\)，即**各分量平方和再开根号**。

**2）欧氏距离 ‖u−v‖₂（问「两个点差多远」）**  
- 先算**差向量**：\(u-v = [3-0,\,4-2] = [3,2]\)。几何上是从 **v 的终点** 指向 **u 的终点** 的位移。  
- 再对差向量取 L2 范数：\(\|u-v\|_2 = \sqrt{3^2+2^2}=\sqrt{9+4}=\sqrt{13}\approx 3.60555\)，约 **3.61**。  
- **含义**：在二维坐标里，点 (3,4) 与点 (0,2) 之间的**直线距离**；在特征空间里可理解为两个样本在各维上数值合起来的「远近距离」（默认各维地位相同，未加权）。

**3）余弦相似度（问「两个箭头方向多像」）**  
- **点积**：\(u\cdot v = 3\times 0 + 4\times 2 = 8\)。同一维相乘再累加。  
- **公式**：\(\cos(\theta)=\dfrac{u\cdot v}{\|u\|_2\,\|v\|_2}\)。这是**夹角 θ 的余弦**（把两向量平移到同一起点后夹角的 cos）。  
- **代入**：\(\|u\|_2=5\)，\(\|v\|_2=2\)，故 \(\cos = 8/(5\times 2)=8/10=0.8\)。  
- **范围**：同向最接近 **1**，垂直为 **0**，反向为 **-1**；**0.8** 表示方向比较接近但未到完全同向。  
- **与长度的关系**：若把 u 换成 **ku**（\(k>0\)，只拉长不转向），则  
  - \( (ku)\cdot v = k\,(u\cdot v) \)，\(\|ku\|_2=k\|u\|_2\)，分子分母各多一个 **k**，**余弦不变**。所以**余弦对「整体缩放」不敏感**，更适合比「语义方向」；**欧氏距离**会随缩放改变。

**本回合小结**  
「先看 `shape`；向量用点积把多维合成一个标量；比远近用距离，比方向用余弦。」

### 回合 3（2026-05-12）：§8～§9 —— 矩阵运算与表格到 `X`、`y`

**§8 矩阵运算（和向量运算不是一回事）**  
- **转置 `A.T`**：行列互换，`(m,k)` → `(k,m)`。常见用途：对齐矩阵乘法维度、实现「行/列」视角切换。  
- **矩阵乘法 `A @ B`（NumPy 推荐）**：第 `(i,j)` 元 = `A` 第 `i` 行与 `B` 第 `j` 列的**点积**。要求 **`A.shape[1] == B.shape[0]`**，结果 `(A 行数, B 列数)`。  
- **逐元素乘 `A * B`（Hadamard）**：形状通常需相同或可广播，**不做**行·列点积累加。混淆 `@` 与 `*` 是新手高频 bug。  
- **记忆**：想「线性组合、一层网络、批量特征变换」→ 想 `@`；想「两个矩阵同一位置相乘」→ 想 `*`。

**§9 从表格到矩阵**  
- **一行** = 一个样本；**一列** = 一个特征（本课约定）。  
- 整张特征表 → 矩阵 **`X`**，`X.shape = (样本数, 特征数)`。  
- **目标变量**（要预测的数或标签）常记 **`y`**，形状多为 `(样本数,)` 或 `(样本数, 1)`。  
- 后续 **scikit-learn**、**PyTorch DataLoader** 都建立在「`X` 与 `y` 行对齐」的习惯上。

### 回合 4（2026-05-12）：§10～§11 —— 概率与数据分布

**§10 概率基础（工程直觉）**  
- **概率**：0～1，描述**不确定性**；分类模型输出的「属于某类 0.82」可理解为模型对该类的主观置信（是否校准另说）。  
- **条件概率 `P(A|B)`**：在 B 成立前提下 A 的可能性；监督学习常被视为学 **P(标签 | 特征)**（形式随任务而变）。  
- **均值 / 方差 / 标准差**：描述数据中心与波动；**标准化特征**、发现异常、对比两批数据都要用。

**§11 数据分布（训练前必做）**  
- **先看直方图 / 分位数**：是否偏斜、长尾、离群点；类别是否极度不平衡。  
- **不必正态**：业务数据多为偏态；了解分布才能选**损失、采样、变换**（如对数变换应对长尾）。  
- **原则**：分布不熟就少写复杂模型，否则难以解释指标好坏。

### 回合 5（2026-05-12）：§12～§15 —— 导数、损失、优化、最小训练闭环

**§12 导数与梯度**  
- **导数**：一个参数微微动，损失**变化多快**、往哪边变。  
- **梯度**：多个参数时，每个参数的偏导组成的向量；**负梯度方向**是局部上让损失下降最快的方向（在合适步长下）。

**§13 损失函数**  
- **损失**：预测与真实的差距；回归里常见 **MSE**（平方误差平均），可导、大误差惩罚更重。  
- **训练目标**：找参数使（训练集上的）**平均损失**尽量小。

**§14 梯度下降**  
- **更新**：`θ ← θ - η ∇L`，`η` 为学习率。  
- **学习率过大**：震荡或发散；**过小**：慢。

**§15 线性回归串讲**  
- 模型 `ŷ = wx + b`，用 MSE，对 `w,b` 求梯度并迭代更新。这是理解 **反向传播** 的最小完整故事：**前向算预测 → 算损失 → 算梯度 → 更新参数**。

**本回合小结（口播）**  
训练 = 选一个**可微的损失**衡量「猜错多少」+ 用**梯度**告诉每个参数往哪拧 + 用**学习率**控制一步迈多大，循环直到损失足够小或达到轮数。**深度学习**只是换成更大的参数集和更长的「前向图」，这条骨架不变。

### 概念答疑：矩阵乘法、embedding、样本特征

**1. 矩阵乘法（Matrix Multiplication）是什么**  
两个矩阵 `A` 与 `B` 能做乘法（记作 `A @ B` 或 `AB`）的前提是：**A 的列数 = B 的行数**。  
若 `A` 形状为 `(m, k)`，`B` 为 `(k, n)`，则乘积形状为 **`(m, n)`**。

**2. 怎么运算（定规则）**  
结果矩阵第 `i` 行、第 `j` 列的元素 = **`A` 的第 `i` 行** 与 **`B` 的第 `j` 列** 对应位置相乘再**全部相加**（即向量**点积**）。

**手算小例**（2×2 乘以 2×2）：

```text
A = [ 1  2 ]    B = [ 5  6 ]
    [ 3  4 ]        [ 7  8 ]

(第1行·第1列) = 1*5 + 2*7 = 19
(第1行·第2列) = 1*6 + 2*8 = 22
(第2行·第1列) = 3*5 + 4*7 = 43
(第2行·第2列) = 3*6 + 4*8 = 50

A @ B = [ 19  22 ]
        [ 43  50 ]
```

工程含义：常把「一批特征」左乘「一层权重矩阵」，得到下一层线性组合结果（再接激活函数等）。

**3. Embedding 是什么**  
把**离散的、不好直接计算的东西**（词、句子、用户 ID、商品 ID）通过模型映射成**固定长度的实数向量**（如 768 维、1536 维），这个向量就叫该对象的 **embedding**。  
用途：比较语义远近（余弦相似度）、检索、聚类。同一模型下，**相近含义**通常对应**方向更接近**的向量（不绝对，依赖训练）。

**4. 样本特征举例**  
- **结构化表格（一条样本 = 一行）**  
  - 样本：某用户；特征向量示例 `[近7天登录次数, 近30天下单数, 累计消费金额(元), 年龄]` → `[3, 2, 1250.5, 28]`。  
  - 样本：某房源；特征 `[面积㎡, 卧室数, 房龄年, 距地铁 km]` → `[89, 3, 5, 1.2]`。  
- **与 embedding 对比**：上表特征是**人定义好的列**；文本「帮我退单」没有天然 5 列数字，要先经模型变成一条长向量，那条才是 **embedding**。

**5. Embedding 数值小例（示意，维数仅为说明）**  

假设把下面三句文字送进**同一个**句子 embedding 模型，输出 **4 维**向量（真实模型常见 768/1536 维，这里只是教学用假数字）：

| 文本           | 示意 embedding（4 维） |
|----------------|-------------------------|
| 「我要退单」   | `[0.82, -0.11, 0.05, 0.33]` |
| 「申请退款」   | `[0.79, -0.09, 0.08, 0.31]` |
| 「今天天气好」 | `[0.12, 0.71, -0.45, 0.02]` |

前两行语义都偏「售后/退款」，向量在数值上**更接近**（余弦相似度会较高）；第三行话题无关，与任一句的相似度通常**更低**。  
**注意**：具体数字由模型与训练数据决定；本表仅说明「**离散文本 → 定长向量，用于比较远近**」。

**6. 矩阵乘法是否可交换（学员答疑）**  

- 一般 **`A @ B ≠ B @ A`**：交换后参与点积的行列角色变了，结果通常不同。  
- 学员补充「除非矩阵里所有数字都一样」：那是一种**很强的特例**（两矩阵同形且元素全相同时确实可交换），但工程中更常见的可交换特例是**单位矩阵**与自身相乘等，不必记全。掌握 **一般不可交换** 即可。
