# 第 5 课：无监督学习、特征工程与模型评估

## 0. 本课阅读方式

本课连接第 3、4 课的监督学习流程，进一步讲三个关键主题：

```text
无监督学习：没有标签时如何发现数据结构
特征工程：如何把原始数据变成模型能学习的输入
模型评估：如何判断模型或特征处理是否有效
```

学习本课时先不要急着写代码。你需要优先理解：

```text
没有 y 时，模型还能学什么？
聚类和分类有什么区别？
降维和特征选择有什么区别？
为什么特征工程经常比换模型更重要？
为什么评估指标必须和业务目标绑定？
```

【核心】无监督学习不是“没有目标”，而是没有人工标签；它的目标通常是发现结构、压缩表示、辅助理解数据。

## 0.1 概念讲义：无监督学习、特征工程与评估术语详解

### 0.1.1 第 5 课到底在学什么

第 4 课的监督学习依赖 `X` 和 `y`：

```text
X：特征
y：标签
```

第 5 课中的无监督学习通常只有：

```text
X：特征
```

没有人工标注的答案 `y`。

因此，无监督学习不直接学习“输入到答案”的映射，而是尝试发现：

- 哪些样本相似。
- 数据是否可以分成几组。
- 哪些维度最重要。
- 高维数据能否压缩到低维。
- 是否存在异常样本。

【重点】无监督学习常用于探索数据、辅助分析、生成特征，而不是直接替代监督学习。

### 0.1.2 Unsupervised Learning 无监督学习

无监督学习是指训练数据没有标签，模型只根据输入特征发现数据结构。

例子：

| 场景 | 无监督学习目标 |
|---|---|
| 用户分群 | 根据行为把用户分成不同群体 |
| 商品分组 | 根据属性把商品聚成相似类别 |
| 异常检测 | 找出和大多数样本不同的点 |
| 文档主题探索 | 发现文档集合中的主题结构 |
| 降维可视化 | 把高维数据压缩到 2D/3D 观察 |

【核心】无监督学习没有标准答案，因此评估通常更依赖业务解释和后续验证。

### 0.1.3 Clustering 聚类

聚类是把相似样本分到同一组。

例如用户分群：

```text
高消费高活跃用户
低消费高活跃用户
沉默用户
新用户
流失风险用户
```

聚类和分类的区别：

| 对比 | 分类 | 聚类 |
|---|---|---|
| 是否有标签 | 有 | 没有 |
| 学习目标 | 学会预测已定义类别 | 自动发现相似分组 |
| 输出含义 | 类别有明确业务定义 | 需要人工解释 |
| 示例 | 是否流失 | 用户自然分群 |

【易错】聚类得到的 cluster 0、cluster 1 没有天然业务含义，需要你分析每组特征后命名。

### 0.1.4 KMeans

KMeans 是常见聚类算法。

它的核心思想是：

```text
先指定 K 个簇
不断调整簇中心
让每个样本归到最近的簇中心
```

关键术语：

- `K`：簇数量。
- `cluster center`：簇中心。
- `inertia`：样本到簇中心距离的总和，常用于观察聚类紧凑程度。

【重点】KMeans 必须提前指定 K。K 的选择不是纯技术问题，也要看业务可解释性。

### 0.1.5 DBSCAN

DBSCAN 是基于密度的聚类方法。

它不需要提前指定簇数量，而是根据样本密度发现簇。

适合：

- 簇形状不规则。
- 存在噪声点。
- 不想预先指定簇数量。

关键参数：

- `eps`：邻域半径。
- `min_samples`：形成核心点所需的最少样本数。

【重点】DBSCAN 对参数和特征尺度敏感，通常需要先做标准化。

### 0.1.6 Dimensionality Reduction 降维

降维是把高维数据压缩到低维。

用途：

- 可视化。
- 去除冗余。
- 降低计算成本。
- 缓解高维稀疏问题。
- 作为后续模型输入。

例如：

```text
100 个特征 -> 2 个主成分
```

【核心】降维不是简单删除列，而是把信息重新压缩表达。

### 0.1.7 PCA

PCA 是主成分分析，一种常见线性降维方法。

它寻找数据中方差最大的方向，把原始特征投影到新的主成分上。

关键术语：

- `principal component`：主成分。
- `explained variance`：解释方差。
- `explained variance ratio`：每个主成分解释的数据变化比例。

【重点】PCA 生成的是新特征，不是从原始特征中选几列。

【易错】PCA 是特征提取，不是特征选择。

### 0.1.8 Feature Engineering 特征工程

特征工程是把原始数据转换成更适合模型学习的特征。

它包括：

- 缺失值处理。
- 类别编码。
- 标准化和归一化。
- 时间特征提取。
- 文本向量化。
- 统计特征构造。
- 交叉特征构造。
- 特征选择。

【核心】传统机器学习项目中，特征工程往往比换模型更重要。

### 0.1.9 Feature Selection 特征选择

特征选择是从已有特征中挑选一部分特征。

目标：

- 减少噪声。
- 降低模型复杂度。
- 提升训练速度。
- 增强可解释性。

【重点】特征选择输出的是原始特征子集。

### 0.1.10 Feature Extraction 特征提取

特征提取是把原始数据转换成新的特征表示。

例如：

- 文本转 TF-IDF。
- 图片转特征向量。
- PCA 生成主成分。
- Embedding 生成语义向量。

【重点】特征提取输出的是新特征，不一定能直接对应原始字段。

### 0.1.11 Preprocessing 预处理

预处理是建模前对数据做清洗和转换。

常见预处理：

- 缺失值填充。
- 异常值处理。
- 标准化。
- 归一化。
- One-Hot 编码。
- 文本向量化。

【核心】预处理必须在训练流程中正确隔离训练集和测试集，避免数据泄漏。

### 0.1.12 Standardization 标准化

标准化把特征转换为均值约为 0、标准差约为 1。

适合：

- KMeans。
- KNN。
- SVM。
- 逻辑回归。
- 神经网络。
- PCA。

【易错】基于距离或梯度的模型通常对特征尺度敏感。

### 0.1.13 Normalization 归一化

归一化通常把数据缩放到固定范围，例如 0 到 1。

它和标准化不同：

```text
标准化：关注均值和标准差。
归一化：关注范围缩放。
```

### 0.1.14 One-Hot Encoding

One-Hot 编码把无序类别转成多个 0/1 特征。

例如：

```text
城市 = 北京 / 上海 / 深圳
```

变成：

```text
city_beijing
city_shanghai
city_shenzhen
```

【易错】不要把无序类别简单编码成 1、2、3，否则模型可能误以为有大小关系。

### 0.1.15 Model Evaluation 模型评估

模型评估是判断模型是否有效。

监督学习评估：

- 有真实标签，可以计算 MAE、RMSE、Accuracy、F1 等。

无监督学习评估：

- 没有标准答案，常结合聚类指标、可视化、业务解释和后续任务验证。

【核心】评估指标必须服务于业务目标，而不是只追求数字好看。

### 0.1.16 Silhouette Score 轮廓系数

轮廓系数用于评估聚类效果。

它考虑：

- 样本和同簇样本是否接近。
- 样本和其他簇是否分离。

取值大致在 -1 到 1：

```text
越接近 1，聚类越清晰。
接近 0，边界模糊。
小于 0，可能分错簇。
```

【重点】轮廓系数只是参考，不能替代业务解释。

【核心】PCA 是特征提取，不是特征选择。

【核心】特征工程决定模型能看到什么信息。

【核心】预处理流程必须防止数据泄漏。

【重点】KMeans 的 K 需要结合指标和业务解释选择。

【重点】无监督学习评估更依赖业务解释。

【易错】聚类编号没有天然含义，需要后续分析命名。

## 3. 无监督学习详解

无监督学习只有输入数据，没有标准答案。

例如用户数据：

```text
登录次数、订单数、消费金额、投诉次数、浏览时长
```

我们不知道每个用户应该属于哪类，但可以让算法根据行为相似性分群。

【工程经验】无监督学习常用于业务探索：先发现群体，再由业务人员解释群体含义。

## 4. 聚类详解

### 4.1 聚类的业务价值

聚类适合回答：

```text
用户能否自然分成几类？
商品是否存在相似组合？
文档是否有主题结构？
异常样本是否远离主要群体？
```

### 4.2 KMeans 工作直觉

KMeans 的过程：

```text
指定 K
随机初始化 K 个中心
把每个样本分配到最近中心
重新计算中心
反复迭代直到稳定
```

【重点】KMeans 基于距离，因此特征尺度会明显影响结果。

### 4.3 如何解释聚类结果

聚类后不要只看标签：

```text
cluster = 0
cluster = 1
```

要看每组特征均值：

```text
cluster 0：高消费、高活跃、低投诉
cluster 1：低消费、低活跃、高投诉
cluster 2：新用户、消费少、登录多
```

然后给业务命名：

```text
高价值用户
流失风险用户
新手观察用户
```

## 5. 降维与 PCA

高维数据难以直接观察。PCA 可以把多个特征压缩到少数主成分。

典型用途：

- 把高维数据降到 2 维画图。
- 降低噪声和冗余。
- 加快后续模型训练。

【易错】PCA 后的主成分通常不再有直接业务字段含义。

## 6. 特征工程详解

特征工程决定模型能接触到什么信息。

例子：

```text
原始字段：total_amount, order_count
构造特征：avg_order_amount = total_amount / order_count
```

这个新特征比原字段更直接表达用户消费习惯。

### 6.1 常见特征工程类型

| 类型 | 示例 |
|---|---|
| 数值处理 | 标准化、归一化、分桶 |
| 类别处理 | One-Hot、目标编码 |
| 时间处理 | 小时、星期、月份、是否周末 |
| 统计特征 | 最近 7 天订单数、平均客单价 |
| 文本特征 | TF-IDF、Embedding |
| 交叉特征 | 投诉率、转化率、复购率 |

【核心】特征工程是把业务理解转成模型可用数字。

## 7. 模型评估与特征评估

评估不仅评估模型，也评估特征处理是否有效。

你需要问：

1. 加入新特征后模型是否提升？
2. 提升是否稳定？
3. 是否引入泄漏？
4. 是否增加过多复杂度？
5. 是否更难解释？

【重点】不是所有新特征都有价值，特征也需要验证。

## 8. 代码辅助示例

### 8.1 KMeans 用户分群

```python
import pandas as pd
from sklearn.cluster import KMeans
from sklearn.preprocessing import StandardScaler

df = pd.DataFrame({
    "login_days": [30, 28, 5, 3, 20, 2],
    "order_count": [20, 18, 1, 0, 8, 1],
    "total_amount": [5000, 4500, 100, 0, 1600, 80],
})

X = df[["login_days", "order_count", "total_amount"]]
X_scaled = StandardScaler().fit_transform(X)

model = KMeans(n_clusters=2, random_state=42, n_init="auto")
df["cluster"] = model.fit_predict(X_scaled)

print(df.groupby("cluster").mean())
```

你要理解：

```text
这里没有 y。
cluster 编号需要业务解释。
标准化会影响距离计算。
```

### 8.2 PCA 降维

```python
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler

X_scaled = StandardScaler().fit_transform(X)
pca = PCA(n_components=2)
X_pca = pca.fit_transform(X_scaled)

print(pca.explained_variance_ratio_)
```

你要理解：

```text
PCA 输出新坐标。
explained_variance_ratio_ 表示主成分解释了多少信息。
```

## 9. 常见误区

### 9.1 把聚类当分类

聚类没有标签，分类有标签。

### 9.2 聚类结果不解释

cluster 0、1、2 没有天然业务意义。

### 9.3 PCA 当成特征选择

PCA 是生成新特征，不是选择原字段。

### 9.4 预处理泄漏

在全量数据上 fit 标准化器再拆分训练测试，会泄漏测试集信息。

### 9.5 只看指标不看业务

无监督学习尤其需要业务解释。
